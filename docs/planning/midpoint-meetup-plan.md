# 약속 중간지점 추천 앱 — 기획 최종안

> 기획팀장 최종 결정 (ROUND 3) · 2026-06-09
> 창의기획자 제안서 + 비평기획자 분석서 종합 판정

---

## 결론 요약

**최종 판정: 조건부 GO — 단, "예약 커미션 = 1순위 수익원"이라는 창의팀 전제는 기각한다.**

웹 조사 결과 비평팀의 두 가지 '치명' 리스크가 데이터로 확인됐다. (1)네이버 예약은 예약만 쓰면 수수료 0%이고 결제를 묶어도 PG 수수료(2.31~3.63%)를 떼면 마진이 거의 남지 않아 — '예약 커미션'은 MVP의 1차 수익원이 될 수 없다. (2)국내 회식 문화는 8년 새 46% 감소했고 가구주 67%가 불황 시 외식비를 1순위로 줄인다 — B2C 오프라인 약속 빈도는 줄어드는 추세이며, 저빈도 리텐션 문제는 시간이 갈수록 악화된다.

따라서 **창의팀의 "웹앱 풀스택 바이럴" 실행 전술은 채택하되, "예약 커미션으로 먹고산다"는 수익 가설은 폐기**한다. MVP는 비평팀 권고대로 **'중간지점 + 그룹 투표 확정' 한 가지**로 좁힌다(예약·정산은 딥링크 연결만, 자체 커미션 모델 아님). 그리고 **개발 착수 전 3개 검증 게이트를 반드시 통과**해야 하며, 게이트 통과 양상에 따라 **B2C 바이럴 vs B2B 회식 SaaS 중 어느 쪽으로 무게를 실을지** 6주 차에 재판정한다.

## MVP 범위 결정 (가장 중요한 판정)

### 핵심 긴장에 대한 판정
> **창의(풀스택 4기능 바이럴) vs 비평(중간지점+예약으로 좁히고 제휴 선검증/B2B 여지)**

**판정: 비평의 '좁혀라'를 채택하되, 좁히는 축을 비평과 다르게 잡는다.**

비평은 "중간지점 + **예약**"으로 좁히라 했지만, 웹 조사상 예약은 수익이 안 나고 제휴 영업 장벽만 높다(비평 스스로도 "제휴 영업장벽 높음"이라 인정). 그래서 **예약을 MVP 핵심 축에서 뺀다.** 대신 창의팀이 2순위로 둔 **'그룹 투표 + 자동 확정'을 1순위로 끌어올린다** — 이것이 (a)카톡 바이럴 루프를 만들고 (b)API 비용이 거의 안 들며 (c)조율자의 진짜 페인('핑퐁 종결')을 직접 푼다.

### ✅ MVP에 넣는 것 (v0.1, 0~6주)
| 기능 | 근거 |
|---|---|
| **다자간 중간지점 추천** (주소 N개 → 최적 역/동네 + 장소 후보 3~5곳 카드) | 제품의 존재 이유. 핵심 페인 직격. |
| **카톡 링크 그룹 투표 + 자동 확정** (앱 설치 없이 웹뷰 투표 → 과반 시 카톡 확정 메시지) | 바이럴 루프 + 조율자 페인('핑퐁') 종결 + API 비용 제로. 창의 2순위를 1순위로 승격. |
| **예약·정산은 '딥링크 패스스루'만** (카카오/네이버 예약·토스 정산 링크를 버튼으로 연결, 자체 결제·커미션 없음) | 사용자 편의는 주되 영업·개발·수익 부담은 지지 않음. 비평의 "기능만 무거워진다" 경고 반영. |

### ❌ MVP에서 빼는 것 (이유 명시)
| 뺀 것 | 이유 |
|---|---|
| **자체 예약 커미션 수익 모델** | 네이버 예약 0% 수수료 + PG 수수료 차감 시 마진 미미(웹 조사 확인). 제휴 가맹점 20곳 선영업 부담 과다. → 게이트 1 통과 시에만 v0.3에서 재검토. |
| **자체 더치페이/정산 엔진** | 토스·카카오페이가 이미 무료로 제공(비평 지적). 딥링크로 충분. 자체 구현은 무거움만 추가. |
| **선호 학습·개인화 추천** | 저빈도라 데이터 안 쌓임(비평 지적, 데이터 검증됨). 품질 의문. v2 이후. |
| **날씨/행사 필터** | nice-to-have. MVP 페인과 무관. |
| **네이티브 앱 출시** | 첫 배포는 웹앱(창의 채택). 심사 없이 빠른 검증 + 카톡 바이럴에 웹앱이 최적. 앱은 v1.0. |

**한 줄 요약**: MVP = **"중간지점 + 카톡 투표 확정"** 두 축. 예약·정산은 링크만. 수익화는 게이트 통과 후 결정.

## 차별화 / 해자

1. **유일하게 '확정'까지 한다** — 카카오/네이버 지도는 장소를 '보여주지만', 다자간 중간지점 → 투표 → 확정을 **한 플로우로 끝내주지 않는다.** 그룹 약속 조율은 그들의 핵심 KPI가 아니다(창의 지적, 타당). 우리는 이 빈틈만 정밀 타격.
2. **바이럴 루프가 곧 해자** — 투표 링크를 받은 멤버 = 잠재 신규 조율자. 약속 1건당 N명에게 제품이 노출되는 구조적 획득 채널. (단, 이것이 리텐션이 아니라 '신규 획득'임을 직시 — 아래 리텐션 설계 참조.)
3. **공평성 알고리즘의 신뢰** — '모두에게 공평한 몫'을 눈에 보이게(각자 이동시간 표시) 제시해 "왜 거기냐"는 논쟁을 종결. 단순 거리중앙이 아닌 **이동시간·환승 가중 공평지수**가 차별 포인트(게이트 2에서 입증 필수).

## 수익모델 + 리텐션 설계 (저빈도 정면 대응)

### 저빈도 문제에 대한 솔직한 인정
비평이 옳다. 오프라인 약속은 본질적으로 저빈도이고, B2C DAU 기반 광고·구독 모델은 이 제품에 맞지 않는다. **"개인이 매일 켜는 앱"을 목표로 하지 않는다.** 대신 두 가지로 우회한다:

### 리텐션 설계
1. **세그먼트 우회**: 일반 사용자가 아니라 **'약속을 자주 잡는 조율자/총무'**를 핵심 페르소나로 삼는다. 이들에겐 월 2~4회의 페인이 반복된다.
2. **획득 ≠ 리텐션 구분**: 바이럴 투표 링크는 **신규 획득** 채널이지 리텐션이 아니다. 리텐션 훅은 (a)약속 히스토리("우리 팀 강남 3회") (b)재모임 시 "지난번 멤버 그대로 다시" 원클릭 (c)정기 모임 리마인드.
3. **궁극의 리텐션 = 계약(B2B)**: 개인 리텐션이 구조적으로 약하다면, **기업 구독 계약으로 리텐션을 대체**한다(비평 핵심 통찰). 회식·팀빌딩 장소를 자주 잡는 건 개인보다 '회사 총무'다.

### 수익모델 (우선순위 재정렬 — 창의의 3층을 재배치)
| 순위 | 모델 | 비고 |
|---|---|---|
| **1순위 (게이트 통과 후)** | **B2B 팀/회사 구독 SaaS** | 회식·워크숍 장소 조율 + 정산 리포트 + 법인카드 연동. 구독료로 API 비용 전가 + 리텐션을 계약으로 해소. 시장 근거 확인됨(팀 운영비 인당 연 120만원). |
| **2순위** | 예약 커미션 (제휴 가맹점 직계약 시에만) | 네이버·카카오 경유 0% 한계. **자체 가맹 직계약 또는 노쇼보증·프리미엄 슬롯 모델일 때만** 수익. 게이트 1 결과에 종속. |
| **3순위 (장기)** | 익명·집계 이동/모임 패턴 데이터 | 충분한 트래픽 확보 후. 프라이버시 정책 선결(Zenly 교훈: 데이터만으로는 못 산다). |

> **첫 6주는 수익화하지 않는다.** 검증이 목적. 매출 압박으로 무거운 예약·결제를 조기 탑재하는 실수를 금지한다.

## 검증 게이트 (개발 착수 전 통과 관문)

> 비평의 '필수 전제조건 3가지'를 **개발 GO/NO-GO를 가르는 관문**으로 재구성. 게이트 미통과 항목은 해당 기능을 출시 범위에서 제외하거나 피벗 트리거로 작동.

### 🚦 게이트 1 — 수익 경로 실재성 (수익화 결정 게이트)
- **통과 기준**: 아래 중 **하나 이상** 충족
  - (a) 예약 커미션이 실제 수익이 되는 가맹점 직계약/노쇼보증 모델 파트너 **10곳+ 구두 합의**, 또는
  - (b) B2B 구독에 관심 있는 **법인 3곳+ 인터뷰에서 "유료로 쓰겠다" 확인**
- **미통과 시**: 예약 커미션을 수익 로드맵에서 **완전 제외**, B2B 게이트로 전환. (창의의 예약 커미션 단독 의존을 차단하는 안전장치)

### 🚦 게이트 2 — 추천 품질 우위 (제품 존재가치 게이트)
- **통과 기준**: 블라인드 사용자 테스트 20명에서 **모이라 추천 ≥ 카카오맵 수동검색을 60% 이상이 선호** (특히 '공평성·이동시간' 항목)
- **미통과 시**: 알고리즘 재설계 후 재테스트. 통과 못 하면 **제품 자체 NO-GO** (차별점이 없으면 존재 이유가 없음).

### 🚦 게이트 3 — 단위경제 흑자 시뮬레이션 (지속가능성 게이트)
- **통과 기준**: MAU 1만 / 5만 / 10만 구간별 **(API 비용 + 인프라) vs (예상 수익)** 시뮬레이션에서 5만 구간 내 흑자전환 경로 존재. OSM 혼용·캐싱 적용한 비용 모델 포함.
- **미통과 시**: 비용 구조(OSM 전환율, 캐싱율) 재설계 또는 B2B 단가 모델로 수익 측 보강.

> **3개 게이트 동시 통과 시에만 v0.1 풀개발 착수.** 게이트 2는 필수(미통과=프로젝트 중단), 게이트 1·3은 '어느 수익 경로로 갈지'를 가르는 분기.
