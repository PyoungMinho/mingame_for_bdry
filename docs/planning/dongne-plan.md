리서치의 전제 교정을 그대로 승계한다. 이 프로젝트는 "하루 10만 방문 사업"이 아니라, **3일 비용만 태우는 데일리 트래픽 실험이자 허브 전략의 1번 타자**다. 법적으로 유일하게 깨끗한 후보(공공 행정구역 경계 데이터)이고, 운영 무인화(250문제 사전 큐잉)가 되며, '어제의 동네' 해설 텍스트 층이 애드센스 요구 콘텐츠와 SEO 롱테일을 자동 생산한다. 서비스명은 카톡에서 자랑·비교로 불리기 쉬운 **"동네고수"**로 확정, 슬러그는 예약어를 피한 **`/dongne`**. 코어는 날짜 기반 결정론적 시드로 서버 없이 전 유저 동일 문제를 출제하고, 오답마다 haversine 거리·8방위 화살표·근접도%만 노출한다(🟩🟨 색블록 그리드 영구 금지). 30일 시점 직전 7일 평균 **DAU 500 게이트**를 통과하지 못하면 즉시 손절한다.

## 2. 코어 게임 루프

### 2-1. 출제 규칙 — 날짜 기반 결정론적 시드 (서버리스, 전 유저 동일)
- **데이터 소스**: `southkorea/southkorea-maps`(KOSTAT 출처) 시군구(municipalities) GeoJSON/TopoJSON. **빌드 타임 오프라인 스크립트**(`scripts/dongne/build-regions.ts`, 런타임 의존성 0 — d3-geo/topojson-simplify는 devDep로 스크립트에서만, 산출물만 커밋)로 250개 시·군·구 정규화 → (1)실루엣 SVG path, (2)중심점 centroid(lat/lng), (3)행정표준코드, (4)표기명·별칭 사전계산. **런타임 단순화 금지**(원본 지오메트리 수 MB 번들 = LCP 2.5s 예산 위반).
- **페이로드 2분할(정답 스포·성능 이중 목적)**: ① `manifest.json` = 250 × `{code, name, aliases[], centroid, bbox}`(≈수십 KB, 자동완성·채점 필수라 초기 로드 OK) ② `silhouettes/{code}.json` = 지역별 정규화 viewBox `d` path(지역당 1~5KB) — **오늘 1개만 온디맨드 fetch**, 250개 전량 초기 번들 금지("소스 보기" 전량 덤프 차단).
- **큐 고정 + 단일 진실 소스**: 250개를 빌드 타임 고정 시드로 1회 셔플 → `QUEUE[250]` 정적 배열 확정(런타임 랜덤 0). **dayIndex(KST)·QUEUE·EPOCH 계산은 `src/lib/dongne/queue.ts` 단일 모듈**로 두고 게임·아카이브·OG·ping이 전부 import(불일치 시 해설이 실제 정답과 어긋남 → 최우선 회귀 테스트). KST는 **고정 UTC+9 산술**(`toLocaleString`/로컬 타임존 금지, 한국 DST 없음).
- **오늘의 문제 인덱스** (KST 기준, UTC 아님 — 한국 유저 자정 동기화):
  ```
  EPOCH = 2026-08-01 (Asia/Seoul 00:00)  // 런칭일로 확정
  dayIndex = floor((now_KST_midnight − EPOCH) / 1day)
  gameNo   = dayIndex + 1                 // 표시용 "#123"
  today    = QUEUE[dayIndex % 250]
  ```
- 서버 상태 0. 브라우저가 날짜만으로 같은 문제를 계산 → 전 유저 동일. 250개 소진 후 wrap-around되나 `gameNo`는 계속 증가시켜 회차 표기 유지(시즌2에서 큐 교체·확장).
- **안티치트는 v1 비목표(명문화)**: 순수 클라 결정론 시드는 정답을 암호학적으로 숨길 수 없다(Wordle과 동일한 내재 속성). v1은 서버 리더보드가 없어 커닝은 커닝한 사람의 재미만 깎을 뿐 → 비목표로 확정. 완화 3종만 적용: ① 정답 확정(성공/6패) 전까지 정답 name/centroid를 DOM·OG·alt에 렌더 금지 ② 오늘 실루엣만 온디맨드 fetch(전량 번들 금지) ③ QUEUE는 코드 배열로만 보관("다음 정답 목록" 친절 구조 노출 금지).

### 2-2. 시도 6회 · 힌트 공식
매 오답 추측마다 **거리 + 방향 + 근접도** 세 값을 텍스트로 노출(색블록 그리드 금지).

- **거리 (haversine, km)** — 추측 지역 centroid ↔ 정답 centroid:
  ```
  R = 6371 km
  dLat = lat2−lat1 ; dLng = lng2−lng1  (라디안)
  a = sin²(dLat/2) + cos(lat1)·cos(lat2)·sin²(dLng/2)
  distance = 2R·atan2(√a, √(1−a))   → 정수 km 반올림
  ```
- **방향 (8방위 화살표)** — 추측 → 정답 bearing:
  ```
  θ = atan2( sin(dLng)·cos(lat2),
             cos(lat1)·sin(lat2) − sin(lat1)·cos(lat2)·cos(dLng) )
  bearing = (θ_deg + 360) mod 360
  8방위 매핑: N ↑ / NE ↗ / E → / SE ↘ / S ↓ / SW ↙ / W ← / NW ↖
  (각 45° 구간, 22.5° offset)
  ```
  → **주의**: 방향 화살표(↗)와 숫자 이모지(1️⃣)는 NYT가 방어하는 🟩🟨 **색상 타일 그리드가 아니므로 트레이드드레스 무관**. 색블록만 영구 금지.
- **근접도 %**:
  ```
  MAX_DIST = 700 km   // 대한민국 대각 최대(제주~고성 ≈ 690km) 상단
  proximity = round( max(0, 1 − distance / MAX_DIST) × 100 )
  정답이면 100%
  ```
- 남은 시도 수 표시, 6회 소진 시 정답 공개 + 해설 링크.

### 2-3. 정답 처리
- 입력: **지명 자동완성**(250개 시·군·구명 + 별칭). 판정은 표기명 문자열이 아니라 **행정표준코드 일치**로.
- 정답 시: 🎯 + 시도 분포 갱신 + 결과 공유 카드 + '어제의 동네' 해설 CTA.

### 2-4. 시·군·구 표기 규칙 (동명 지역 처리)
- 기본 표기: `시군구명`. **동명이 존재하면 `시군구명(시도 축약)`** 병기.
  - 예: `고성군(강원)` · `고성군(경남)`, `남구(부산)` · `남구(대구)` · `남구(광주)` 등.
- 광역시 자치구는 동명 충돌이 잦으므로 자치구 전부 `구명(광역시)` 형태를 자동완성에 노출.
- **세종특별자치시**는 하위 시군구가 없으므로 시 자체를 1문제로 취급.
- 자동완성에서 동명은 **둘 다 각각 노출**하고 선택 시 코드로 구분 → 오판정 방지.

- 네이티브 앱, 다국어, 결제/구독.

### 5-1. DAU 집계 방법 (게이트 진실 소스 = Supabase ping)
> 백엔드팀장 교정: **Vercel Web Analytics 무료 티어에 게이트를 걸지 않는다.** Hobby 이벤트 상한(월 2,500)이 DAU 500×PV×30일에서 즉시 초과·샘플링되고, 공유 Vercel 프로젝트라 전 라우트 혼합 집계 → `/dongne`만 깔끔히 못 뽑는다.
- **게이트 진실 소스(확정)**: `POST /api/dongne/ping` + Supabase 테이블 `dongne_dau(day_kst date, anon_id text, created_at timestamptz, PK(day_kst, anon_id))`. 레포에 이미 wired된 유일 영속 저장소(Supabase, `/pae` 운영 중) 재사용 → **신규 env 0 · 신규 의존성 0**(deploy.yaml env-drift 게이트 자동 통과).
- **클라이언트**: localStorage 익명 `anon_id`(uuid) 1회 발급 + `dongne:ping:{YYYYMMDD}` 플래그로 **KST 하루 1인 1회만** POST.
- **서버**: `day_kst`를 **서버에서 UTC+9로 스탬프**(클라 스푸핑 방지·날짜 버킷 권위화), `insert ... on conflict do nothing`으로 (day, anon_id) 유니크 dedupe.
- **판정 쿼리**: `SELECT day_kst, count(distinct anon_id) FROM dongne_dau GROUP BY 1` → 직전 7일 이동평균 ≥ 500.
- **DAU 정의 = 게임 페이지(`/dongne`) 진입 시 ping = 일별 순 anon_id.**
- Vercel Web Analytics는 **보조 sanity check로만**(게이트 판정 미사용). package.json 변경을 피하기 위해 v1은 **미도입/이연**(필요 시 `dongne/layout.tsx`에만 격리 마운트).

## 6. 성공 / 킬 기준

- **게이트(킬 스위치)**: 런칭 후 **30일 시점, 직전 7일 평균 DAU ≥ 500**이면 계속. 미만이면 **즉시 손절**(코드·데이터·해설 자산은 허브 전략용으로 보존, 도메인 유지 여부는 별도 판단).
- **측정 기준**: 5-1의 DAU 정의(순 방문자, 7일 이동평균)로 판정. 시딩 스파이크 당일은 제외하고 잔존 기준.
- **보조 지표(참고, 게이트 아님)**: D1 리텐션, 공유 클릭률, 평균 세션 길이, "우리 동네" 뱃지 도달률.
- **광고수익은 게이트 무관** — 리서치 전제(2)대로 승인 1~2개월 수익 0이 기본값. DAU 게이트를 광고보다 먼저 순수 트래픽으로 판정한다.
