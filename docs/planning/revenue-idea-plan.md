# 변형 시험지 생성 SaaS "문제팩토리(가칭)" 기획 최종안

> 기획팀 ROUND 3 — 기획팀장 최종 결정 / 2026-06-05
> 사장(PM) GO 판단용 확정 기획서. 확정 즉시 빌드 착수 가능 범위로 작성.

---

## 결론 요약

최종 확정 아이디어는 **한국 사교육 강사용 "변형 시험지·문제은행 자동 생성 SaaS"**다. 강사가 자신의 PDF/한글(HWP) 기출문제나 교재를 업로드하면, Claude API가 **같은 개념의 변형 문제(숫자·보기·지문 변형, 난이도 조절, 해설 자동 생성)**를 대량 생성하고, 출력은 **인쇄용 시험지 PDF + 문제은행 축적**으로 떨어진다. 창의기획자의 1·2순위(#8 구글리뷰, #3 유튜브요약)는 둘 다 **빌드 마찰(외부 API 승인) + 단발성 AI 래퍼**라는 이중 결함으로 탈락시켰고, 비평기획자가 단단하다고 본 #1·#6도 각각 무료 OSS 포화/Mailchimp 무료티어로 방어막이 약해 기각했다. 최종안은 비평기획자의 핵심 루브릭(B2B·고단가소수·워크플로우임베드·데이터락인)을 **4개 모두 충족**하면서, 입력이 강사 본인의 파일이라 **외부 승인 의존이 0**이고, 수 주 내 1인+Claude Code로 빌드 가능하다. 기존 도구(비바샘 출제마법사, 유사변형 시험지 업체)가 이미 존재한다는 사실은 **시장 수요와 지불 의사가 이미 검증되었다는 강력한 신호**이며, 그것들이 AI 이전 세대(수작업 DB·타이핑 기반)라 LLM 워크플로우로 비집고 들어갈 명확한 틈이 있다.

## 1. 창의 vs 비평 충돌 판정

### 충돌 ①: #8 구글리뷰 자동응답기 (창의 1순위 vs 비평 "AI복제·리텐션 취약")
**판정: 비평 승. 추가로 빌드 마찰로 즉시 탈락.**
- 비평의 "GPT 래퍼" 지적이 정확하다. "리뷰에 답글 써주기"는 ChatGPT에 붙여넣으면 끝나는 단발 작업이라 전환비용(락인)이 거의 없다.
- 결정타는 **빌드 마찰**: 핵심 가치(리뷰 자동 수집·게시)는 **구글 비즈니스 프로필 API + OAuth 심사**에 묶인다. 심사는 수 주~수개월 걸릴 수 있어 "수 주 내 출시" 제약과 정면 충돌. 심사 회피 시 수동 복붙으로 전락 → 차별화 소멸.
- 보강 가능성: 낮음. 외부 의존을 빼면 제품의 본질이 사라진다. **탈락 확정.**

### 충돌 ②: #3 유튜브 강의 요약+퀴즈 (창의 2순위 vs 비평 "리텐션 매우 취약·AI복제")
**판정: 비평 승. 추가로 데이터 수급 경로가 막혀 탈락.**
- 비평의 "단발성·즉시 복제" 지적이 정확하다. 강의 하나 요약하면 끝, 재방문 유인이 약하다.
- 결정타: 조사 결과 **YouTube 공식 Data API는 자막/트랜스크립트를 프로그래밍 방식으로 제공하지 않는다.** 즉 (a)제3자 스크래핑 도구 의존(ToS·차단 리스크) 또는 (b)별도 Compliance Audit 승인이 필요. "외부 승인·규제 의존 최소화" 제약 위반.
- 보강 가능성: 낮음. 데이터 수급 자체가 불안정한 토대 위에 올라간다. **탈락 확정.**

### 충돌 ③: 비평이 인정한 #1 라이선스 / #6 리타게팅 이메일
**판정: 루브릭은 통과하나 방어막 부실로 둘 다 기각.**
- #1 라이선스: 조사 결과 **무료 OSS 도구(Syft, Trivy, GitHub Dependency Graph)가 SBOM 니즈의 ~80%를 커버**하고 FOSSA·Snyk·Sonatype 등 강자가 포진. 인디 개발자는 지불 의사가 낮고(창의기획자도 단가 4천원 지적), 콜드스타트 채널은 있으나 "공짜로 충분"한 시장이라 유료 전환이 어렵다.
- #6 리타게팅 이메일: Mailchimp/Klaviyo 무료티어와 정면 경쟁. 차별화 어렵고 이메일 발송은 도메인 인증·스팸 정책 등 별도 운영 부하.

### 결론적 통찰
창의팀의 매력(콜드스타트 채널이 분명함)과 비평팀의 규율(락인·고단가)을 **동시에** 만족하려면, **"입력이 고객 본인의 데이터 → 외부 API 불필요 → AI로 즉시 변환 → 출력물이 반복 업무를 대체 → 쌓을수록 락인"** 구조여야 한다. 후보 8개 중 이 4박자를 외부 승인 없이 만족하는 것은 **#3을 재정의한 "자료 입력형 콘텐츠 생성"**이되, 유튜브가 아니라 **강사 본인 PDF**를 입력원으로 바꾼 형태다. 글로벌 일반 AI 퀴즈 시장은 Kahoot·Quizizz가 PDF→퀴즈를 이미 무료 제공해 포화이므로, **한국 사교육 "변형 시험지" 니치로 좁혀** 방어막을 만든다.

## 2. 채택한 아이디어 (재정의·재가격)

**#3을 폐기하지 않고 "입력원 교체 + 시장 재정의"로 재설계**했다.
- 입력: 유튜브 영상 → **강사 본인의 기출/교재 PDF·이미지·HWP 텍스트**
- 시장: 글로벌 일반 학습자 → **한국 사교육(보습학원·과외·재수학원) 강사·원장**
- 출력: 단발 요약 → **반복 사용하는 변형 시험지·문제은행(자산 축적형)**

이렇게 하면 비평 루브릭 4개를 모두 충족한다:
- (a) B2B ✅ — 강사·학원 대상
- (b) 고단가 소수 ✅ — 월 2.9만~9.9만원 × 50~200사
- (c) 워크플로우 임베드 ✅ — 매주 반복되는 "시험지/숙제지 출제" 업무에 박힘
- (d) 데이터 락인 ✅ — 강사가 쌓은 문제은행·변형 이력은 해지 시 손실되는 전환비용

## 3. 최종 기획서

### ① 제품 한 줄 정의 + 좁힌 니치
> **"강사가 기출 PDF 한 장을 올리면, 같은 유형의 변형 시험지 30문항과 해설을 5분 안에 만들어 인쇄까지 끝내주는, 한국 사교육 강사 전용 문제 생성 SaaS."**

- **좁힌 니치**: 1차 타깃은 **중·고등 보습학원 및 1인 과외/공부방 강사 중 "내신 변형·숙제지·주간 클리닉 시험"을 매주 직접 만드는 사람**. (수능 모의고사 제작 같은 거대 출판 시장은 제외 — 1인이 못 이긴다.)
- 과목 우선순위: **영어(어법·어휘 변형) → 수학(숫자·조건 변형) → 국어/사회/과학**. 영어·수학이 변형 수요가 가장 크고 변형 자동화가 LLM에 가장 잘 맞는다.

### ② 타깃 · 핵심 페인포인트
| 타깃 | 페인포인트 |
|---|---|
| 보습학원 강사 | 매주 반 별로 다른 숙제지·주간시험을 손으로 변형 제작 → **주당 5~10시간 소모**. 시험지 한글 편집·수식 입력이 노가다. |
| 1인 과외/공부방 원장 | 학생 수준별로 난이도 다른 문제를 따로 만들어야 함. 문제 재탕하면 학생이 답을 외워버림(변형 필수). |
| 학원 원장 | 강사마다 제각각인 시험지 퀄리티·포맷을 표준화하고 싶음. 강사 퇴사 시 문제은행이 같이 사라짐. |

**핵심 한 줄 페인**: "같은 개념인데 숫자·보기만 바꾼 문제를 매주 수십 개 손으로 만드는 반복 노동" → LLM이 가장 잘하는 일.

### ③ 월 천만원 수익 모델 (보수적 전환 가정)

**가격 티어 (B2B, 세금계산서 발행 → PG 의존 최소화)**

| 티어 | 월 요금 | 대상 | 핵심 한도 |
|---|---|---|---|
| 라이트 | 29,000원 | 1인 과외·공부방 | 월 300문항 생성 / 문제은행 1과목 |
| 프로 | 59,000원 | 보습학원 강사 | 월 1,500문항 / 전 과목 / 변형 무제한 / PDF 브랜딩 |
| 학원(팀) | 149,000원 | 학원 단위 | 강사 5인 / 공유 문제은행 / 원장 대시보드 |

**보수적 시나리오 (목표 = 월 1,000만원 = 10,000,000원)**

```
프로  59,000원 × 120사 = 7,080,000원
학원 149,000원 ×  15사 = 2,235,000원
라이트 29,000원 ×  30사 =   870,000원
                        ─────────────
                 합계 = 10,185,000원  ✅
```
- 총 유료 고객 **165사**. B2B SaaS 마이크로 기준 충분히 도달 가능한 규모(비평기준 50~200사 범위 내).
- **보수성 근거**: 평균 단가 약 6.2만원으로, 저단가 다수(250~500명)에 의존하지 않는다. 한국 사교육 종사자 수십만 명 모집단 대비 165사는 침투율 극저점. 무료체험(14일) → 유료 전환 5%만 가정해도 누적 도달 가능.
- **업셀 경로**: 라이트 → 프로(과목·한도 확장), 프로 → 학원(강사 늘면 자연 전환). 락인된 문제은행이 클수록 상위 티어로 끌려 올라감.

### ④ 수 주 내 빌드 가능한 MVP 기능 범위 (외부 승인 의존 0)

**기술 스택**: Next.js + Claude API (기존 환경 그대로). DB는 Postgres(Supabase) 또는 SQLite로 시작.

**MVP 핵심 경로 (외부 심사 없는 것만)**
1. **파일 업로드 → 텍스트 추출**: PDF/이미지 업로드 → 텍스트·수식 추출(PDF 파싱 + 필요 시 Claude 비전으로 이미지 문제 읽기). HWP는 1차에서 "텍스트 붙여넣기" 입력으로 우회(HWP 직접 파싱은 후순위).
2. **변형 생성 엔진**: 원문항 → Claude API로 "동일 개념·다른 숫자/보기/지문" 변형 N개 생성 + 정답 + 해설. 난이도 슬라이더(하/중/상).
3. **시험지 조판·PDF 출력**: 생성 문항을 시험지 레이아웃에 자동 배치 → 인쇄용 PDF 다운로드(워터마크/학원명 브랜딩). 정답·해설 분리 출력.
4. **문제은행(락인 코어)**: 생성·검수한 문항을 강사 계정에 저장·태깅(단원·난이도·유형). 재사용·재변형·즐겨찾기. **이것이 해지 방어막.**
5. **계정·구독 관리**: 회원가입/로그인, 무료체험 카운터, 사용량 미터링. **결제는 MVP에서 수동 인보이스(세금계산서)로 처리** → 사업자등록·PG 심사가 출시를 막지 않게 함. (유료 고객 생기면 그때 토스페이먼츠/스트라이프 연동.)

**MVP에서 의도적으로 뺀 것 (전부 외부 승인·복잡도 사유)**
- 카카오 알림톡(사업자·템플릿 심사) → 이메일/웹 알림으로 대체
- 결제 PG 실연동 → 수동 세금계산서로 시작
- HWP 네이티브 파싱·완벽 수식 렌더링(LaTeX) → 1차는 텍스트/이미지 기반, 수식은 점진 고도화
- LMS·온라인 응시 기능 → 출력은 "인쇄 PDF"에 집중(강사 실제 워크플로우)

> MVP는 **"PDF 넣으면 변형 시험지 PDF 나온다"** 한 줄 가치에 모든 빌드를 집중한다. 이 경로에 외부 심사가 하나도 없다는 점이 본 안의 최대 강점.

### ⑤ 차별화 · 방어막 (데이터 락인)
1. **한국 사교육 도메인 특화**: Kahoot·Quizizz는 "객관식 일반 퀴즈"지 "내신 변형 시험지"가 아니다. 한국 영어 어법 변형, 수학 조건 변형, 시험지 조판(2단·정답표)은 현지 강사 워크플로우에 맞춰야만 쓸 만하다 → 글로벌 도구가 안 들어오는 해자.
2. **데이터 락인(핵심)**: 강사가 수개월간 검수·축적한 **개인 문제은행·태그·변형 이력**은 해지 시 통째로 손실. 쌓일수록 전환비용 증가 → 리텐션 방어.
3. **출력물 품질 = 진입장벽**: 인쇄해서 학생에게 바로 나눠줄 수 있는 조판 완성도(정답·해설 분리, 학원 브랜딩)는 GPT 복붙으로 재현 불가. "래퍼"라는 비판을 **출력물 워크플로우 완성도**로 방어.
4. **AI 복제 방어**: 단순 프롬프트 래퍼가 아니라 (입력 파싱 → 도메인 특화 변형 → 조판 → 문제은행 축적)의 **엔드투엔드 워크플로우**가 가치. 복제하려면 이 전체를 한국 강사 디테일까지 맞춰야 함.

### ⑥ 6개 리스크별 대응
| # | 비평 살상 리스크 | 본 안의 노출도 | 대응 |
|---|---|---|---|
| 1 | **콜드스타트** | 단면시장(공급=강사뿐)이라 양면시장 즉사 리스크 없음 | 강사 커뮤니티(아래 GTM)에서 출시 전 예비고객 50명 확보, 첫 30명 무료 평생/할인 |
| 2 | **리텐션 붕괴** | 매주 반복되는 출제 업무 + 문제은행 축적으로 구조적 방어 | 주간 사용 트리거(매주 시험지 필요) + 락인 데이터 + "재변형" 재방문 유인 |
| 3 | **1인 번아웃** | CS·온보딩 부하 위험 | FAQ봇·온보딩 시퀀스 자동화, B2B라 고객 수 적어(165사) 관리 가능, 셀프서브 설계 |
| 4 | **CAC > LTV** | 고단가(평균 6.2만원)로 LTV 높음 | 무료 채널(커뮤니티·콘텐츠 SEO) 우선, LTV:CAC 3:1 유지, 연결제 할인으로 LTV↑ |
| 5 | **AI 래퍼 포화** | "변형 생성" 자체는 복제 가능 | 도메인 특화 + 조판 + 문제은행 락인으로 워크플로우 해자 구축(⑤ 참조) |
| 6 | **규제·결제** | 학생 개인정보·결제 심사 | **MVP는 강사 자료만 다루고 학생 개인정보 미수집**, 결제는 세금계산서 수동으로 PG 심사 회피 |

추가 고유 리스크 — **저작권**: 강사가 올리는 기출이 출판사 저작물일 수 있음 → 제품은 "사용자가 권리 있는 자료만 업로드" 약관 명시 + 결과물은 "변형 생성물"로 위치, 원문 재배포 기능은 넣지 않음(개인 출제 보조 도구로 한정).

### ⑦ 초기 100명 확보 GTM 채널
타깃이 **디지털 채널에 모여 있다**(비평이 콜드스타트 설계 가능 조건으로 요구한 바로 그것).
1. **강사 커뮤니티 침투**: 네이버 카페(학원·과외·공부방 운영자 카페), 오르비/수만휘 강사 게시판, 영어·수학 강사 단톡방. 출시 전 "변형 시험지 자동화 베타 테스터" 모집 → 50명 예비고객.
2. **콘텐츠 SEO/유튜브**: "내신 변형문제 5분 만에 만드는 법", "강사 시험지 제작 시간 줄이기" 등 페인 키워드 블로그·쇼츠 → 검색 유입.
3. **무료 미끼(Lead Magnet)**: "기출 PDF 올리면 변형 10문항 무료" 랜딩 → 체험 후 가입 전환.
4. **학원 직접 영업**: 동네 보습학원 원장 대상 데모(학원 티어 149,000원 타깃). 1사가 강사 5명 → 효율적.
5. **인플루언서 강사 제휴**: 팔로워 있는 인기 강사 1~2명에게 무료 제공 → 후기·추천(레퍼럴).

목표: 출시 전 50명 대기 → 출시 4주 내 무료체험 100명 → 전환으로 첫 유료 20~30사 확보.

### ⑧ 출시 후 4주 로드맵
> 전제: 빌드(MVP 코어)는 확정 즉시 착수, 아래는 **출시 시점 이후 4주**.

| 주차 | 목표 | 핵심 작업 |
|---|---|---|
| **W1 출시** | 영어 1과목 + 변형생성 + PDF출력 + 문제은행 + 무료체험 라이브 | 베타 대기자 50명에게 오픈, 온보딩 시퀀스 가동, 첫 피드백 수집 |
| **W2 전환** | 수학 과목 추가, 조판 품질 개선 | 무료체험 → 유료 전환 푸시(만료 알림), 첫 세금계산서 발행, CS FAQ봇 배치 |
| **W3 확산** | 학원(팀) 티어 + 공유 문제은행 출시 | 학원 원장 데모 영업 시작, 커뮤니티 콘텐츠 2차 배포, 레퍼럴 코드 |
| **W4 정착** | 결제 자동화(토스페이먼츠) 연동, 리텐션 점검 | 유료 30사 목표 점검, 이탈 인터뷰, 국어/사회 과목 백로그 착수, 가격·한도 튜닝 |

**4주 성공 기준(보수)**: 무료체험 100명 / 유료 전환 20~30사 / MRR 150만~250만원 → 천만원 경로의 1차 검증.

## 4. 백업(차순위) 아이디어

**#5 B2B 소형 제조사 납기추적 SaaS (재설계 조건부)** — 비평 루브릭 4개를 본질적으로 충족(B2B·고단가·워크플로우 임베드·거래처 공유 데이터 락인)하는 가장 단단한 후보. **단, 본 안 대비 후순위인 이유**: (1)핵심 가치였던 카카오 알림톡이 사업자·템플릿 심사라 빠른 출시를 막음 → MVP에서 빼고 웹·이메일 공유로 대체해야 함, (2)현장 온보딩 부하·긴 영업주기로 1인 번아웃 리스크가 더 큼, (3)콜드스타트 채널(제조사)이 디지털에 덜 모여 있어 GTM이 어렵다. **본 안이 GO 불가 판정 시**, #5를 "카카오 제외·웹 공유 링크 기반"으로 재설계해 2순위로 전환한다.
