## 0. Executive Summary (사장 의사결정용 1쪽)

**프로젝트**: 일기 한 편을 AI가 4컷 만화로 변환 → SNS 공유 + 365편 아카이브를 누적하는 습관형 SaaS.

**확정 사항 (팀장 합의 완료)**
- 이미지 생성: **Google Gemini 2.5 Flash Image (Nano Banana) on Vertex AI** 1순위, OpenAI gpt-image-1 2순위. ImageProvider 추상화로 페일오버.
- 프론트: **Next.js 15 + React 19 + Tailwind v4 + TanStack Query v5 + Zustand + react-konva + Satori**.
- 캐릭터 일관성: **아바타 8종 + reference image + 시드 고정 + 얼굴 임베딩 검사**, multi-turn 4컷.
- 한글 처리: 이미지에 한글 절대 미생성, **CSS/SVG 오버레이로 통일**.
- 브랜드: "감성 65% + 키치 35%", 컬러/타이포/로고/아바타 8종/말풍선 4종 디자인 토큰 확정.
- 통합 일정: **디자인 3주 선행 → 프론트 6주 + 백엔드 8주 병렬 (총 11주)**.

**결재 완료 (사장이 PM에게 위임 → 2026-06-03 확정)**
1. 가격 정책: **베이직 ₩9,900 / 프로 ₩17,900 + 크레딧 팩 + 연간(2개월 무료) + 베타 얼리버드 평생 30% 할인**. (옵션 A 기반 + 충동구매·현금흐름 보강)
2. 초기 인프라 예산: **₩20M 린 클로즈드 베타** (대기자 명단·DAU 캡 ~1,000·배치 기본). Go/No-Go 통과 후에만 오픈 베타 예산 ₩60M 집행.
3. 벤더 전략: **Gemini 단독 운영 + ImageProvider 추상화로 OpenAI 즉시 교체 가능하게 코드만 준비** (API는 사용량 과금이라 "동시계약" 추가비용 없음 → 안전성과 저비용 동시 확보).

## 1. 확정된 기술 스택

### 1.1 이미지 생성 API
| 순위 | 제공자/모델 | 단가 | 채택 이유 |
|---|---|---|---|
| 1순위 | **Google Gemini 2.5 Flash Image (Nano Banana) / Vertex AI asia-northeast3** | $0.039/장 | 멀티턴 캐릭터 일관성 공식 지원, 배치 50% 할인, 서울 리전 지연 최소 |
| 2순위 (백업) | OpenAI gpt-image-1 (medium) | 더 비쌈 | Gemini 장애/정책 변경 시 페일오버 |

- `ImageProvider` 인터페이스 추상화 의무 (백엔드팀장 결정).
- 배치 가능한 워크로드(예약 생성, 백필)는 무조건 배치 API 라우팅.

### 1.2 프론트엔드 스택
- **Next.js 15 (App Router) + React 19 + TypeScript strict**.
- 스타일: Tailwind v4 + 디자인 토큰(디자인팀 산출).
- 상태: TanStack Query v5(서버) + Zustand(클라이언트).
- 캔버스: **react-konva** (말풍선 에디터).
- OG/공유 카드: **Satori** (1080×1080 / 1920×1080 / 1080×1920 3종 동시 생성).
- 스트리밍: **SSE + Server Action**, panel 단위 Suspense.

### 1.3 인프라
- 클라우드: **GCP asia-northeast3 (서울)** — Vertex AI 동일 리전.
- 큐: 생성 작업 큐잉 (GPU/비용 가드 + 우선순위 = Pro > Basic > Free).
- 인증/결제: Auth + 결제 모듈 (백엔드 W1~2).
- 관측: 비용 메트릭(장당 원가 실측), 큐 지연, SSE 끊김률, 모더레이션 통과율.
- 저장: 원본 이미지(콜드), 썸네일/공유카드(핫), 일기 텍스트 암호화.

## 2. 핵심 아키텍처 결정 (7개)

1. **이미지 한글 절대 금지** — 텍스트(말풍선·캡션)는 100% CSS/SVG 오버레이. 백엔드는 말풍선 메타데이터(좌표/타입/꼬리방향)만 반환.
2. **아바타 8종 프리셋 + 시드 고정 + reference image multi-turn** — LoRA 미사용. 1컷 생성 후 그 이미지를 reference로 2~4컷 연쇄 생성하여 일관성 확보.
3. **얼굴 임베딩 후처리 검사** — 동일 사용자 4컷 간 얼굴 유사도 임계치 미달 시 자동 재생성(최대 2회).
4. **부분 결과 SSE 스트리밍** — 1컷부터 즉시 노출, 대기 이탈 방어. 프론트는 panel 단위 Suspense.
5. **워터마크 = 프론트 처리** — 백엔드는 tier 정보만 제공, 프론트가 Satori 템플릿 분기(무료=큼, 유료=작음 or 제거).
6. **화풍 4종 캘리브레이션 = 디자인↔백엔드 주2회 미팅 의무** — 골든 프롬프트 + 골든 참조 이미지 세트 유지, 모델/프롬프트 변경 시 회귀 테스트.
7. **모더레이션 2단** — 입력 텍스트(미성년/자해/선정) + 출력 이미지(NSFW/얼굴 유사). 책임 한계 약관 + 미성년자 보호자 동의 플로우.

## 3. 비용 시뮬레이션 + 가격 정책 재검토 결과

### 3.1 백엔드팀장 시뮬레이션 요약 (DAU 3,000)
| 시나리오 | 월 인프라 비용 |
|---|---|
| 즉시생성 100% | ₩29.8M |
| 배치 혼용 | ₩17~20M |

**BEP 달성 조건**: 유료 전환 12%+ **또는** 객단가 ₩9,500+ 필요. 현 정책(₩6,900 Basic / 5% 전환 가정)으로는 미달.

### 3.2 가격 정책 — 최종 확정 (2026-06-03, 사장 위임 결정)

사장이 PM에게 위임 → 옵션 A를 기반으로 **충동구매 캡처(크레딧 팩)와 현금흐름(연간)을 보강**한 안으로 확정.

| 티어 | 월 요금 | 연 요금 | 한도/혜택 |
|---|---|---|---|
| **무료** | ₩0 | — | 1일 1회 미리보기(512px, 워터마크) — 바이럴 유입 엔진 |
| **베이직** | **₩9,900** | ₩99,000 (2개월 무료) | 월 30컷, HD(1024px), 워터마크 제거, 화풍 4종, 아바타 기본 커스텀 |
| **프로** | **₩17,900** | ₩179,000 (2개월 무료) | 월 150컷(공정사용), 우선 생성, 아바타 고급 커스텀, PDF/고해상도, 신규 화풍 우선 |
| **크레딧 팩** | 1컷 ₩2,000 / 5컷 ₩7,900 / 12컷 ₩15,900 | — | 구독 거부 충동형(여행·기념일) 캡처. 무기한 보유 |
| **베타 얼리버드** | — | — | 베타 기간 구독자 **평생 30% 할인** (초기 전환 데이터 + LTV 확보) |

**근거**
- 베이직 ₩9,900은 ₩10,000 미만 차밍 프라이스 + 단가 압박 방어. 프로는 ₩20,000 미만으로 캐주얼 사용자 진입장벽 완화.
- **크레딧 팩**: 창의팀이 짚은 "특별한 날 결제 욕구 폭발" 순간을 구독 강요 없이 흡수. 충동 매출 = 이 제품의 두 번째 수익 기둥.
- **연간 + 얼리버드**: 캐시번 구간에서 선결제 현금 확보 + 초기 락인.
- 블렌디드 ARPU 추정: 베이직 60% / 프로 25% / 크레딧 15% 가정 시 ≈ **₩11,000** → BEP 객단가 기준(₩9,500) 충족.
- v1.1 이후 옵션 C(광고로 무료 +1편) 보조 도입 여지 유지.

### 3.3 비용 가드레일 (수치 게이트)
- 장당 실원가 **$0.05 이하** 유지(배치 활용 포함).
- 사용자당 일일 생성 한도 hard cap = tier별 한도 × 1.2 (안전 마진).
- 어뷰징 의심 시 사용자 단위 자동 throttling + 알림.

### 3.4 베타 예산 — 린 클로즈드 베타 ₩20M (2026-06-03 확정)

사장 지시("예산은 적게"). ₩60M은 **오픈 베타(DAU 3,000 스케일)** 가정치였음. 베타는 **클로즈드·캡**으로 운영해 번을 1/3로 절감하고, 동시에 "성공할수록 빨리 파산" 리스크를 원천 차단한다.

| 항목 | 3개월 합계 | 비고 |
|---|---|---|
| 이미지 생성 (DAU 캡 ~1,000, 배치 기본) | ₩7~8M | 무료 미리보기 512px + 캡으로 폭주 차단 |
| 인프라(서버/DB/Redis/CDN) | ₩1.8M | Cloud Run + Neon + Upstash + R2 |
| 법무 자문(일회성) | ₩3M | 약관 + 미성년자 플로우 + 상표 검색 |
| 도구/관측/도메인 | ₩1.5M | Sentry, 모니터링, 도메인 |
| 예비비 | ₩6M | 버퍼 |
| **합계** | **≈ ₩20M** | |

- 운영 방식: **대기자 명단 + 초대제**, DAU 하드캡 ~1,000. 캡 도달 시 신규 초대 일시 중단.
- **스케일업 트리거**: §8.3 출시 전 게이트(BEP 시뮬·법무·알파 만족도) 통과 시에만 오픈 베타 예산 ₩60M 집행 결재.

## 5. 통합 개발 일정 (총 11주)

```
W1  W2  W3  W4  W5  W6  W7  W8  W9  W10 W11
[디자인 3주]
    [프론트 6주, 디자인 W1 종료 직후 토큰/와이어 받아 시작]
    [백엔드 8주, 디자인 W1 종료 직후 시작]
                                        [통합 QA 2주]
```

### 5.1 디자인 (W1~W3, 선행)
- W1: 디자인 토큰 + 와이어프레임 → **프론트/백엔드 착수 트리거**
- W2: 코어 화면 + 일러스트(아바타 8종, 말풍선 4종, 화풍 골든세트)
- W3: 폴리시 + 핸드오프

### 5.2 프론트 (W2~W7)
- W2 디자인시스템 → W3 일기입력 → W4 4컷결과/SSE → W5 말풍선에디터 → W6 공유카드 → W7 워터마크/QA

### 5.3 백엔드 (W2~W9)
- W2~3 인프라/Auth/결제 → W4 LLM 분절 → W5 Gemini 통합/캘리브레이션 → W6 큐 → W7 quota/어뷰징 → W8 말풍선 메타 → W9 부하/관측

### 5.4 통합 (W10~W11)
- W10 통합 QA + 모더레이션 회귀 + 부하 테스트
- W11 알파 테스트(20명) + 버그픽스 + 출시 준비

> **align 결정**: 프론트는 백엔드 W4(Gemini 통합) 완료를 W5(말풍선 에디터)까지 mock으로 우회 → W5 후반에 실제 API 통합. 백엔드 W4 종료 시점이 통합 마일스톤.
