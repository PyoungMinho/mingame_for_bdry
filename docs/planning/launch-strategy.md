# 문제팩토리 런칭 전략 최종 기획안

> 작성: 기획팀장 (ROUND 3 최종 결정)
> 일자: 2026-06-09
> 전제: MVP 빌드·검증 완료 (테스트 174 통과, 10페이지 정상 빌드). 공개 배포 전.
> 목표: 월 매출 1,000만원 / 1인 운영 (사장=PM 본인)

## 결론 요약

광고 선배포는 **하되**, 그것을 메인 수익 전략으로 삼지 않는다. 핵심은 **"광고는 트래픽·신뢰 증명용, 진짜 매출은 강사 B2B"**라는 투트랙이다. 사장이 느끼는 "사업자등록=빡세다"는 심리적 장벽은 실측 검증 결과 **사실과 다르다** — 토스페이먼츠가 사업자등록을 5분 내 대행하고, 간이과세 실질 세율은 1.5~4%에 불과하다. 따라서 **사업자등록을 미루는 것이 가장 비싼 실수**라는 비평팀 판단을 채택하되, 사장의 부담을 줄이기 위해 **"광고 배포로 먼저 트래픽/수요를 눈으로 확인한 뒤, 그 데이터를 들고 사업자등록을 결심하는"** 2~3주 완충 구간을 설계했다. 로그인은 강제하지 않는 선택형, 스케줄러는 풀기능 대신 D-day 2줄 훅으로 축소한다.

## 1. 3가지 질문에 대한 최종 결정

| # | 질문 | 결정 | 한 문장 결론 |
|---|------|------|------------|
| Q1 | 학생용만 광고로 먼저 배포하고 강사 유료는 미룰까? | **조건부 YES → 단, "미룬다"가 아니라 "2~3주만 병행 준비"** | 광고 선배포는 무해하고 권장하지만, 강사 유료를 "미루는" 것이 아니라 **광고로 트래픽 증명하는 동안 사업자등록·PG를 병행 신청**해 3주 뒤 강사 구독을 연다. |
| Q2 | 로그인을 넣는 게 더 편할까? | **조건부 YES (선택형만, 강제 절대 금지)** | "저장하려면 로그인" 방식의 **선택형 소셜 로그인**은 도입하되, 첫 사용에 회원가입을 강제하면 핵심가치(무마찰)가 죽으므로 Phase 1에는 넣지 않고 Phase 2에 붙인다. |
| Q3 | 수능 스케줄러(플래너)를 추가할까? | **NO (현 단계 반대) → D-day 2줄 훅만 YES** | 열품타식 풀 플래너는 스코프 킬러라 반대하고, **"수능 D-243 │ 오늘 생성한 문제 수"** 초경량 훅 한 줄만 넣는다(개발 반나절). |

### 근거 (검증된 수치 기반)

- **Q1 근거**: 한국 애드센스 RPM은 보수적 $2~낙관적 $5/1,000PV. 월 1,000만원을 광고만으로 내려면 **월 200만 PV(일 6.7만)** 가 필요한데, 이는 콴다급 트래픽으로 광고 단독은 2~3년 과제다. 반면 기존 기획서 가정대로 매출의 75%가 강사 B2B(프로 59K·학원 149K)이므로, **강사 유료를 미루는 것 = 매출의 4분의 3을 미루는 것**. 광고는 "배포해도 무해"하지만 "그것만으로 천만원"은 산술적으로 불가능하다.
- **Q2 근거**: 강제 로그인 시 이탈 40~60%는 무마찰 UVP를 정면으로 부순다. 그러나 localStorage만으로는 기기 변경 시 기록이 날아가 **강사 유료 전환(서버 동기화 필요)**이 막힌다. 따라서 "강제는 안 하되, 저장·동기화를 원하는 사람만 로그인"이 정답.
- **Q3 근거**: 창의·비평 양측이 모두 풀 플래너 반대로 일치. 플래너는 별도 제품 수준의 스코프이고, 핵심가치(문제 생성)와 무관해 리소스를 분산시킨다. 단 "D-day"는 수능생 동기부여 훅으로 retention에 기여하면서 개발 코스트가 거의 없다.

## 3. "지금 당장(이번 주) 배포 가능한 최소 안전 버전" 체크리스트

### A. 사장이 직접 해야 할 일 (개발 무관, 지금 시작)
- [ ] **AdSense 신청** — `/study`에 콘텐츠 충분히 채운 뒤 구글 애드센스 가입 신청 (심사 1~2주, 지금 신청 시작이 핵심)
- [ ] **(병행) 토스페이먼츠 가입 알아보기** — 사업자등록 미보유 시, 토스페이먼츠 "사업자등록 바로신청"으로 5분 내 개인사업자 등록 가능. 빡세다는 인식과 달리 통합 처리됨
- [ ] **간이과세 선택** — 사업자등록 시 간이과세자 신청(연매출 1.04억 미만 / 실질 세율 1.5~4% / 부가세 연 1회). 부담 최소화
- [ ] **PG 정기결제 심사 신청** — 토스페이먼츠 정기결제(구독) 상점 ID로 심사 신청 (평균 5영업일, 정기결제 포함 1~2주). Phase 2 결제에 필요하므로 지금 큐에 넣기
- [ ] **강사 수요 검증(선택)** — 크몽/탈잉에 "수능 변형문제 제작" 리스팅으로 사업자등록 전 수요 측정 가능(검증 용도 한정)

### B. 개발이 바로 붙일 수 있는 것 (사업자등록과 무관, 즉시 착수)
- [ ] **D-day 2줄 훅** — "수능 D-243 │ 오늘 생성 N문제" 표시 (반나절, localStorage 카운트)
- [ ] **`/exam` 베타 배너** — "베타 기간 무료" 안내 + 강사 이메일 수집 폼 (반나절)
- [ ] **AdSense 코드 삽입** — `/study`에 광고 슬롯 배치 (심사 통과 후 활성화)
- [ ] **레이트리밋 점검** — 이미 추가된 API 비용가드 정상 작동 재확인 (라이브 모드 토큰비용 폭주 방지)
- [ ] **만 14세 미만 약관 노출** — 가입/이용 약관에 "만 14세 미만 이용 제한" 명시 (이미 약관 있으므로 문구 추가만)

> **핵심**: B(개발)는 **오늘 바로** 시작 가능. A(사장 행정)는 **AdSense 신청 + 토스페이먼츠 알아보기**만 이번 주에 하면 됨. 사업자등록을 "지금 당장 완료"할 필요는 없으나, **신청 큐에는 이번 주에 올려야** Phase 2가 3주 뒤 열린다.

## 4. 냉정한 기대값

### 수익원별 현실적 월매출 추정 레인지
| 수익원 | 보수적 | 낙관적 | 전제 |
|--------|--------|--------|------|
| **광고(AdSense)** | 월 5만 | 월 30만 | 일 1,000~3,000 PV 기준. RPM $2~$5. **천만원과는 거리가 멈** |
| **2,900원 광고제거 구독** | 월 10만 | 월 50만 | 활성 사용자의 1~3% 전환 가정 |
| **강사 프로 59K** | 월 60만 | 월 300만 | 유료 강사 10~50명 |
| **학원 149K** | 월 150만 | 월 750만 | 학원 10~50곳 (Phase 3) |

### 월 1,000만원까지의 현실적 타임라인
| 시점 | 누적 월매출 (현실 시나리오) | 비고 |
|------|--------------------------|------|
| 1개월(Phase 1) | 5~30만 (광고만) | 트래픽 데이터 확보 단계 |
| 3개월(Phase 2) | 80~400만 | 구독+강사 프로 가동 시작 |
| 6개월(Phase 3 초입) | 300~700만 | 학원 플랜 초기 도입 |
| **9~12개월** | **800~1,200만 (목표 도달 가능권)** | 학원 B2B가 주 엔진. 단 강사 영업 실행이 변수 |

> **냉정한 진실**: 광고만으로는 12개월 후에도 월 50만 미만에 머문다. 천만원은 **강사·학원 B2B가 만든다**. 그러므로 "광고로 먼저 시작하고 강사를 미룬다"는 전략은 목표 도달을 **6~12개월 지연**시킨다. 광고는 출발 신호, 천만원의 엔진은 B2B다.

## 보류 / 추가 검토 사항
- **PG 최종 선택**(토스페이먼츠 vs 포트원): 정기결제 수수료·심사 속도 실제 견적 비교 후 Phase 2 착수 시 확정
- **강사 영업 채널**: 천만원의 핵심인 학원 B2B를 어떤 경로(직접 영업/제휴/입소문)로 확보할지는 Phase 2 데이터 보고 별도 기획 필요
- **2,900원 구독 가격 민감도**: 실제 전환율은 배포 후 A/B로 검증
