# 프로젝트 방향서 — 직장인 성향 분석기 (Office Archetype)

> STEP 1(기획 회의) 산출물. 기획팀 5인 3라운드 회의 → 팀장 최종 결정. 디자인·개발팀 공용 브리프.
> slug: `office-archetype` · 라우트: `src/app/office-archetype/` · 배포: `project-orsrw.vercel.app/office-archetype`

## 1. 한 줄 정의

직장 생활 패턴 **10문항** → **8개 성향 유형** 분류 + **상성 좋은 동료 유형 매칭**을, 블라인드/팀 카톡에서 퍼지는 **자기확인형 바이럴 테스트**.

## 2. 왜 이 도메인인가 (회의 결론)

- 창의·비평 **양팀이 독립적으로 동일하게 1픽** 으로 선정. 4개 렌즈(트래픽·경쟁해자·수익화·빌드난이도) 중 **kill이 하나도 없는 유일한 아이템**.
- **핵심 차별점 = 콘텐츠가 "고정형 로직"**: 나BTI·밸런스게임류는 매주 새 콘텐츠를 써야 하는 인건비 지옥에 빠지지만, 성향 분석기는 **8유형 매트릭스를 한 번만 잘 설계하면 순수 클라이언트 로직(DB/API 0)으로 무한 재사용**. 솔로가 Vercel 라우트 하나로 만들기에 최적.
- **콜드스타트 채널 실존**: 블라인드(대기업 직장인 10명 중 8명, 30대 37%) + `/pae`(오후의 패)로 이미 검증한 팀메이크 직장인 채널·배포 파이프라인과 정확히 중첩.
- **라벨링 문화 수요 실증**: 나BTI 14.6만 참여 → 2026년 현재도 "나를 라벨 붙이는" 수요 살아있음.

## 3. "하루 10만"의 정직한 재정의 (팀장 honestTake)

단일 슬러그 하나로 하루 10만은 사실상 불가능. 다음 3개 전제로 재정의한다.

1. **스파이크 최고점**이지 baseline 아님 — 대부분의 날은 그 10~30% 수준.
2. **테스트 팩토리 포트폴리오 합산**으로만 지속 — 동일 인프라 위에 분기마다 신규 테스트를 찍어냄. office-archetype = **1번 타자**.
3. **KPI = "트래픽 → 후속 서비스 유입구"** (팀메이크 브랜드/오후의 패·모이라 크로스 유입, CAC 절감). 광고 단독은 월 50~70만원 수준이라 부족.

> 접근 철학: "10만짜리 한 방"이 아니라 **"싸게 여러 번 던져서 한 번 터지면 시리즈로 굳힌다"** 는 확률 게임. 실패해도 이틀이면 다음 타자(빌드 1~2일, DB 0).

## 4. 트래픽 모델 (3단)

1. **콜드스타트**: 8유형을 블라인드 인기글 200개+ 정성분석으로 설계 → 직장인 베타 30~50명 공감률 검증 → 블라인드/에펨/디시 직장인 게시판에 "내 유형 나옴" 후기 시딩. 초기 1~2주 스파이크로 하루 3~5만 목표(보수).
2. **2차 확산 루프**: 결과 카드에 **"상성 좋은 동료 유형" 매칭** → "너랑 나랑 팀 짜면 좋대" 식 팀 단위 재공유 → 1인 완결형을 2~n인 참여형으로 전환, K-factor ≥ 1 지향.
3. **지속화**: 같은 엔진 위에 신규 테스트(재택형·부업형·민감도 HSP 등)를 계속 찍어냄 → 개별 밈의 2~4주 반감기를 포트폴리오가 상쇄.

## 5. 8유형 설계 원칙 (⚠️ 최우선 리스크 게이트)

**정확도가 낮으면 공유는커녕 "이거 별로다" 부정 바이럴로 역전된다** — 4개 렌즈 전부가 반복 경고. 따라서:

- 8유형은 **기존 MBTI 재탕 금지**. 직장 맥락 고유의 축(예: 일 vs 관계, 주도 vs 순응, 안정 vs 도전, 겉맴 vs 진심 등)으로 새로 설계.
- **블라인드 인기글 200개+ 정성분석**을 유형 설계 입력으로 사용(최소 2주 투입).
- **런칭 전 직장인 30~50명 베타로 "내 얘기 맞다" 공감률 검증** — 미달 시 재설계. **이 검증이 배포 게이트.**

## 6. 산출물 요구사항 (디자인/개발 공통)

### 화면 (4개)
1. **랜딩/인트로**: 훅 카피 + "시작하기" (모바일 우선, 3초 안에 뭔지 이해)
2. **질문(10문항)**: 초경량 선택지 UI, 진행률, 뒤로가기, 한 손 조작
3. **결과**: 8유형 중 1개(이름·비주얼·설명·강점/그림자) + **상성 동료 매칭**
4. **공유 카드**: 카카오/인스타 최적화 OG 이미지 **2종(1:1, 9:16)** 원클릭 공유

### 엔진 (개발 핵심)
- **재사용 가능한 "테스트 엔진"**: `질문·유형 데이터를 JSON으로 분리`(하드코딩 금지). 이 엔진이 시리즈 전체 공용 인프라.
- 순수 클라이언트 로직. **DB/API 의존 0** 지향(초기).
- 카톡 공유를 위한 OG 메타 + 결과 이미지 동적 생성.

## 7. 수익화 (KPI 재정의 전제)

- 1차: 디스플레이+네이티브 광고(단독으론 부족).
- 실질: **팀메이크 브랜드/후속 서비스(오후의 패·모이라) 유입구**로 재정의. 결과 페이지에 채용/이직·직장인 B2B 제휴(고단가 CPM), "우리 팀 전체 유형 리포트" 소액 유료/기업 팀빌딩 옵션.

## 8. 첫 빌드 스텝 (팀장 지정)

`src/app/office-archetype/` 라우트 생성 → 콘텐츠 없이 먼저 **테스트 엔진 골격**(10문항 선택 → 매트릭스 판정 → 8유형 결과 → OG 공유 카드 1:1·9:16) 구축. 질문·유형은 JSON 분리. 병행하여 8유형 정성분석 + 베타 공감률 검증.
