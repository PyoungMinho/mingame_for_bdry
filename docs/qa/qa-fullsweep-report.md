## 2. H-1 — 단답형 즉시채점 동치성 (Critical) ✅

### 증상
학생이 **의미상 정답**을 입력해도 표기 차이로 오답 처리:
- 과학 정답 `1.50` ↔ 학생 입력 `1.5` → 오답 (소수점 trailing 0)
- 영어 정답 `decides → decide`(화살표 포함) → 학생이 입력 자체가 불가능
- 국어 정답 = 본문 전체 문장 → 입력 불가능
- 사회 정답 = 사례 문장 → 입력 불가능
- 수학(정수 정답)만 정상 동작

5개 과목 단답형 중 **4개가 사실상 채점 불능** — 기획의 핵심 가치("즉시 채점")가 데모에서 깨져 있었음.

### 근본 원인
1. `study/page.tsx`의 인라인 `normAnswer`가 `.replace(/[.,]/g,"")`로 **소수점을 제거** → `1.50`→`150`, `1.5`→`15` → 불일치.
2. 데모 빌더(`exam-engine.ts`)의 영어/국어/사회 단답 정답이 **타이핑 불가능한 형태**(화살표·전체 문장).

### 수정
- **`src/lib/grading.ts` 신규** — 순수 채점 로직 분리(React 비의존, 단위 테스트 가능):
  - `parseNumeric()` — 순수 숫자만 number, 천단위 콤마/공백 허용, 단위·기호는 null.
  - `answersMatch()` — 양쪽이 숫자면 **수치 동치 비교**(`1.5 === 1.50`), 그 외엔 정규화 텍스트 비교(`decide === " Decide "`).
- **`study/page.tsx`** — 인라인 비교 제거, `answersMatch` 사용.
- **`exam-engine.ts`** — 데모 영어/국어/사회 단답을 **타이핑 가능한 정답**으로 + 문제 발문을 단답형에 맞게 수정:
  - 영어 `decides → decide` → **`decide`** ("올바른 형태를 한 단어로 쓰시오")
  - 국어 (본문 문장) → **`연쇄`** ("○○적 붕괴 — 두 글자를 쓰시오")
  - 사회 (사례 문장) → **`기회비용`** ("…차선책의 가치를 뜻하는 경제 용어를 쓰시오")
  - 과학/수학 발문은 유지(수치 동치 비교로 해결).

### 검증
- **단위 테스트** `tests/study/grading.test.ts` 신규 10케이스: `1.50==1.5`, `6.00==6`, `.67==0.67`, `1,000==1000`, 텍스트 정규화, 경계값(`""`/`-`/`5m/s`/`1.2.3`→null) 전부 통과.
- **데모 빌더 회귀** `engine.test.ts` 3케이스 갱신(`decide`/`연쇄`/`기회비용`) 통과.
- **라이브 E2E**(브라우저): 과학 단답 "질량 2kg·힘 3N → 가속도?"(정답 1.50)에 **`1.5` 입력 → "정답이에요!"** + 점수 +1. 오답 검증: 다른 문항(정답 1.33)에 `9` 입력 → **"아쉬워요. 정답은 1.33"**(점수 미증가). 과대관대(false positive) 없음.

## 3. F-1 — `/favicon.ico` 404 (High) ✅

### 증상
브라우저가 자동 요청하는 `/favicon.ico`가 404. 원인은 문서 `<head>`에 **아이콘 링크가 전혀 없어** 브라우저의 유일한 폴백이 레거시 경로뿐이었기 때문.

### 수정
- **`public/icon.svg` 신규** — ∑ 브랜드 마크(인디고 `#4F46E5` 라운드 사각 + 흰 ∑), 512×512 벡터.
- **`layout.tsx` `metadata.icons`** 추가 → `<link rel="icon" type="image/svg+xml" href="/icon.svg">` + `shortcut` + `apple-touch-icon` 주입.
- **`manifest.ts` `icons[]`** 추가 → PWA 설치 아이콘.

### 검증
- `/icon.svg` → **200**, `/manifest.webmanifest` → **200**.
- `<head>`에 아이콘 링크 3종 정상 주입 확인.
- 실제 페이지 이동 시 브라우저가 `/favicon.ico`를 **자동 요청하지 않음**(SVG 링크로 충족). 네트워크 로그의 유일한 `/favicon.ico` 404는 QA 진단용 수동 `fetch` 1건뿐 — 실사용자 노출 없음.
- (잔여·비기능) 누군가 `/favicon.ico` 경로를 **직접** 요청하면 여전히 404. 완전 제거를 원하면 `src/app/favicon.ico` 바이너리 추가 가능(별도 승인 시).

## 4. M-1 — OpenGraph 공유 이미지 미설정 (Medium) 📋 보고

`layout.tsx`의 `openGraph`에 `images`가 없어 카톡/슬랙/트위터 공유 시 **썸네일이 안 뜸**. 기능 결함은 아니나, "회원가입 없이 무료" 바이럴 유입에 직접적 손해. → **1200×630 OG 이미지 제작 + 등록** 필요(디자인 에셋 작업이라 승인 후 진행 권장).
