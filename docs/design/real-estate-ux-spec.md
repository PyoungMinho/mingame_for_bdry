## 2. 정보 구조 (IA)

### 2.1 화면 계층 맵

```
진짜집 (/)
│
├── [P0] 검색 홈 (/)
│   ├── 신뢰순 정렬 리스트 (기본)
│   ├── 필터 패널 (가격/평수/배지 등급)
│   └── 매물 카드 → 매물 상세
│
├── [P0] 매물 상세 (/listings/[id])
│   ├── 사진 갤러리 (블러 통과본)
│   ├── 신뢰 스코어 패널 (총점 + 5항목 막대 분해)
│   ├── 매물 기본 정보
│   ├── 문의하기 (CTA)
│   └── 신고하기 → 신고 시트
│
├── [P0] 세입자 업로드 게이트 (/verify)
│   ├── Step 1: 본인인증 (PASS)
│   ├── Step 2: 전자동의
│   ├── Step 3: 사진 촬영/선택
│   ├── Step 4: 처리 대기 (폴링)
│   └── Step 5: 결과 확인 (등급 + 점수)
│
└── [P0] 본인인증·전자동의 게이트 (/auth/verify)
    ├── PASS 인증 진입
    ├── 약관·전자동의 확인
    └── 완료 → /verify 복귀
```

### 2.2 네비게이션 구조

```
[하단 탭바 - 3개 핵심 탭]
 [홈/검색]   [업로드]   [내 활동]
    /           /verify    /my

[상단 헤더]
- 로고 (좌) → / 이동
- 검색창 (중앙, 홈에서만 확장형)
- 알림 (우) → 처리 완료·신고 결과 수신
```

네비게이션 원칙:
- 소비자(검색) 경로와 세입자(업로드) 경로는 탭으로 분리 — 역할 혼동 방지
- 인증 게이트(/auth/verify)는 별도 탭 없음 — 업로드 플로우 내에서만 진입
- 신고 시트는 모달/바텀시트 — 별도 탭 없이 상세 화면에서 직진입

## 3. 핵심 사용자 플로우

### 3.1 플로우 A — 소비자: 검색 → 신뢰순 탐색 → 상세 확인 → 문의

```
[진입]
  앱 실행 or 링크 접속
      │
      ▼
[검색 홈 /]
  기본 상태: 신뢰순 정렬 리스트 노출
  Gold 배지 매물이 상단 점령 (서버 사전계산, 프론트 정렬 없음)
      │
      ├─ 필터 적용 (선택) → 가격/평수/배지 등급 필터 → 리스트 갱신
      │
      ▼
[매물 카드 탭]
  카드에서 확인 가능:
  - 신뢰 배지 (Gold/Silver/Unverified) + 총점 숫자 + 자연어 라벨
  - 대표 사진 (블러 통과본)
  - 보증금/월세/평수/위치
      │
      ▼
[매물 상세 /listings/[id]]
  RSC로 렌더링 (로딩 스켈레톤 → 콘텐츠)
      │
      ├─ 사진 갤러리 탐색 (스와이프)
      │
      ├─ 신뢰 스코어 패널 확인
      │   └─ 5항목 막대 분해 → "왜 82점인지" 투명 공개
      │
      ├─ 매물 정보 확인 (층수/관리비/입주가능일 등)
      │
      ├─ [선택] 의심스러우면 → 신고하기 (플로우 C 진입)
      │
      └─ 문의하기 탭 (CTA)
             │
             ▼
         [문의 완료] → 중개사 연결 안내
```

**핵심 UX 결정:**
- 3탭 원칙 준수: 홈 탭(1탭) → 카드(2탭) → 문의(3탭)
- 신뢰순은 기본값 — 사용자가 별도 조작 없이도 인증 매물이 상단에 위치
- 상세 화면은 RSC — SEO + 초기 로딩 성능 모두 확보

### 3.2 플로우 B — 세입자: 본인인증 → 사진 업로드 → 블러 확인 → 등급 획득

```
[진입]
  하단 탭 [업로드] 탭 또는 홈 배너 "방 인증하고 혜택 받기"
      │
      ▼
[Step 0: 매물 연결]
  "어느 매물에 올릴까요?"
  - 주소 검색 또는 QR/링크로 매물 선택
  - 매물 미존재 시 → 중개사에게 등록 요청 안내
      │
      ▼
[Step 1: 본인인증 게이트 /auth/verify]
  ─────────────────────────────────
  PASS 인증 진입 버튼 (1탭으로 완결)
    ↓ PASS 앱 실행 (딥링크)
    ↓ 인증 완료 콜백 수신
  ─────────────────────────────────
  실패 시: "인증에 실패했습니다. 다시 시도해주세요" + 재시도
      │
      ▼
[Step 2: 전자동의]
  - 약관 핵심 3가지만 체크박스 (전문은 접힌 아코디언)
    □ 사진 게시 주체는 세입자 본인임에 동의합니다
    □ AI 자동 블러링 처리에 동의합니다
    □ 게시 후 신고 시 즉시 비공개 전환에 동의합니다
  - IP·타임스탬프·약관버전 불변 로그 서버 기록 (사용자 미노출)
      │
      ▼
[Step 3: 사진 촬영/선택]
  ─────────────────────────────────────────
  [기본 CTA] 앱 내 카메라로 지금 찍기
    → "앱 내 촬영 시 EXIF가 자동 기록되어 인증 점수가 높아집니다"
    → 카메라 권한 요청 → 촬영 화면
       촬영 가이드 오버레이: 방 4방향 + 창문·욕실 권장 안내

  [보조 CTA] 내 사진첩에서 가져오기
    → 앰버 경고 배너: "외부 사진은 EXIF 검증이 어려워 점수가 낮을 수 있습니다"
    → 선택 계속 가능 (강제 차단 아님)
  ─────────────────────────────────────────
  사진 업로드 후 → 진행률 바 표시 (원본은 서버 비공개 버킷에 저장)
      │
      ▼
[Step 4: 처리 대기 (비동기)]
  ─────────────────────────────
  [처리중 상태]
  진행 중... [애니메이션 스피너]
  "AI가 개인정보를 안전하게 블러 처리하고 있습니다."
  "보통 30초~2분 소요됩니다."
  폴링/Realtime 구독으로 서버 결과 수신 (낙관적 UI 없음)
  ─────────────────────────────
  이탈 허용: "나중에 확인하기" → 알림 ON 권유 → 홈으로
      │
      ▼
[Step 5: Before/After 블러 프리뷰]
  ┌──────────────┬──────────────┐
  │   원본 프리뷰  │   블러 처리본  │
  │  (썸네일 미리) │  (공개될 모습) │
  └──────────────┴──────────────┘
  "얼굴·개인정보가 자동으로 가려졌습니다"
  [수동 블러 보정] 버튼 → 추가 블러 영역 지정 가능
  [게시하기] / [다시 찍기]
      │
      ▼
[Step 6: 결과 확인 — 즉시 피드백]
  ┌─────────────────────────────────┐
  │  Gold 획득                     │
  │  실거주 인증 · 82점            │
  │                                │
  │  +35점 세입자 사진              │
  │  +18점 EXIF·생활흔적           │
  │  ────────────────────          │
  │  상단 노출 적용됨               │
  │  이사 확인 인증서 발급 가능     │
  └─────────────────────────────────┘
  [인증서 받기] / [홈으로]
```

**핵심 UX 결정:**
- PASS 인증은 1탭 딥링크 — 4단계 이탈 방지의 핵심
- 앱내 카메라 유도는 강제가 아닌 점수 차등으로 설득
- 낙관적 UI 금지 — 처리 완료 전까지 점수 표시 없음

### 3.3 플로우 C — 신고: 의심 매물 신고 → 테이크다운

```
[진입]
  매물 상세 화면 (/listings/[id]) 하단
  "신고하기" 버튼 탭
      │
      ▼
[신고 바텀시트]
  ┌────────────────────────────────┐
  │ 어떤 문제인가요?               │
  │                               │
  │ ○ 이미 거래된 매물             │
  │ ○ 사진이 실제와 다릅니다       │
  │ ○ 정보가 허위입니다            │
  │ ○ 개인정보가 노출됩니다        │
  │ ○ 기타                        │
  │                               │
  │ [추가 설명] (선택, 최대 200자) │
  │                               │
  │ [신고 제출]                   │
  └────────────────────────────────┘
      │
      ▼
[신고 접수 확인]
  "신고가 접수되었습니다."
  "검토 중에도 매물은 계속 볼 수 있습니다."
  → 신고 즉시: 해당 매물 비공개 전환 (notice-and-takedown)
      │
      ▼
[신고됨 상태 — 비동기 처리]
  매물 상세 화면에 진입 시:
  ┌──────────────────────────────────┐
  │  [신고 검토 중]                  │
  │  이 매물은 신고를 받아 검토 중   │
  │  입니다. 잠시 후 다시 확인해     │
  │  주세요.                        │
  └──────────────────────────────────┘
      │
      ├─ 검토 완료·허위 확인 → 테이크다운 (영구 비공개)
      │   → 신고자에게 앱 알림: "신고하신 매물이 내려갔습니다"
      │
      └─ 검토 완료·정상 판정 → 매물 복구
          → 신뢰 스코어 커뮤니티 항목 소폭 차감 유지 (신고 이력 기록)
```

**핵심 UX 결정:**
- 신고 즉시 비공개 전환 (notice-and-takedown) — 법적 안전 + 사용자 신뢰
- 신고 후 결과 알림 — 신고자가 "처리됐는지" 알 수 있어야 재신고 동기 유지
- 바텀시트 모달 — 별도 페이지 전환 없이 3탭 원칙 내 완결

## 4. P0 4화면 와이어프레임

### 4.1 검색 홈 (/)

```
┌─────────────────────────────────────────┐
│ [로고 진짜집]          [알림 아이콘]    │  ← 헤더 (48px)
├─────────────────────────────────────────┤
│ 🔍  어디서 살 거예요?  [검색창]        │  ← 검색바 (56px)
│  관악구   마포구   [필터▼]              │  ← 지역 퀵탭 + 필터
├─────────────────────────────────────────┤
│ 정렬: [신뢰순▼]   147개 매물            │  ← 정렬 컨트롤 (서버 사전계산)
├─────────────────────────────────────────┤
│                                         │
│  ┌──────────────────────────────────┐  │
│  │ [사진 썸네일]  [Gold 배지 82점]  │  │  ← 매물 카드 (높이 130px)

│  │ 보증금 1,000 / 월 45만          │  │
│  │ 실거주 세입자가 찍음             │  │  ← 자연어 라벨
│  └──────────────────────────────────┘  │
│                                         │
│  ┌──────────────────────────────────┐  │
│  │ [사진 썸네일]  [Silver 배지 67점]│  │
│  │ 마포구 신촌동 · 오피스텔 · 15㎡  │  │
│  │ 보증금 3,000 / 월 65만          │  │
│  │ 현장 인증됨                      │  │
│  └──────────────────────────────────┘  │
│                                         │
│  ┌──────────────────────────────────┐  │
│  │ [사진 썸네일]  [Unverified 31점] │  │  ← 앰버 경고 톤
│  │ 관악구 봉천동 · 원룸 · 10㎡     │  │
│  │ 보증금 500 / 월 38만            │  │
│  │ ⚠ 미인증 매물                    │  │
│  └──────────────────────────────────┘  │
│                                         │
│  [무한스크롤 — 커서 페이지네이션]       │
├─────────────────────────────────────────┤
│  [홈]         [업로드]      [내 활동]  │  ← 하단 탭바 (56px)
└─────────────────────────────────────────┘

※ 배지 규격:
  Gold     = 방패 아이콘 + 초록 + 체크마크    (색+형태 이중코드)
  Silver   = 카메라핀 아이콘 + 노랑           (색+형태 이중코드)
  Unverified = 점선 회색 원 + 앰버 텍스트     (색+형태 이중코드)
※ 카드 내 점수: 숫자 + 미니 도넛 (16px 크기)
```

**영역 우선순위:**
1. 배지 + 자연어 라벨 (신뢰 판단의 1순위 신호)
2. 가격 (전환 결정의 핵심 정보)
3. 면적/위치 (적합성 필터)
4. 사진 썸네일 (3순위 — 상세에서 확인 유도)

## 5. 비동기 상태별 UX 처리

### 5.1 상태 정의

| 상태 | 정의 | 사용자 언어 | 시각 처리 |
|---|---|---|---|
| `pending` | 외부 미검증 (미검증 ≠ 위반) | "보류 중 (외부 확인 대기)" | 회색 빈 막대 + 회색 텍스트 |
| `processing` | 서버 파이프라인 처리 중 | "분석 중..." | 펄스 애니메이션 + 진행 안내 |
| `reported` | 신고됨, 테이크다운 진행 | "신고 검토 중" | 앰버 배경 배너 |
| `taken_down` | 영구 비공개 확정 | "이 매물은 내려갔습니다" | 전체 화면 회색 처리 |
| `verified` | 검증 완료 | 점수 + 자연어 라벨 | 등급별 배지 색+형태 |

## 6. 업로드 이탈 방지 UX 상세

### 6.1 이탈 지점 식별

| 단계 | 이탈 원인 | 이탈률 예상 | 대응 전략 |
|---|---|---|---|
| Step 1: PASS 인증 | "또 인증이야?" 피로 | 높음 | PASS 1탭 딥링크 + 소요 시간 명시 "30초 완결" |
| Step 2: 전자동의 | 긴 약관에 거부감 | 중간 | 3개 핵심만 노출 + "모두 동의" 숏컷 + 전문은 아코디언 |
| Step 3: 사진 촬영 | 카메라 권한 거부 | 중간 | 권한 요청 전 목적 설명 + 권한 거부 시 복구 가이드 |
| Step 3: 사진 선택 | "내 사진으로 하면 안 되나" | 중간 | 강제 차단 아님 — 점수 차이 설명 후 자율 선택 |
| Step 4: AI 처리 중 | 긴 대기 → 앱 이탈 | 높음 | 이탈 허용 + 알림 권유 + 진행률 세분화 |
| Step 5: 블러 결과 | "내 물건 블러됐나?" 불신 | 낮음 | Before/After 비교 프리뷰 + 수동 보정 옵션 |

### 6.2 이탈 방지 세부 설계

#### 단계 진입 전 기대값 설정 (진행 스텝바)

- 스텝바를 항상 상단 고정으로 노출 — 전체 5단계 중 어디에 있는지 인지
- 각 스텝 레이블 하단에 예상 소요시간 표기:
  ```
  1.인증(30초) → 2.동의(1분) → 3.촬영(2분) → 4.처리(2분) → 5.완료
  ```
- 완료된 스텝은 채워진 원 + 체크, 현재 스텝은 강조, 미완료는 비어있는 원

#### 이탈 감지 시 확인 다이얼로그

```
[뒤로가기 또는 X 탭 시]

┌─────────────────────────────────┐
│  지금 나가면 진행이 저장되지    │
│  않습니다.                      │
│                                 │
│  [계속 진행하기]   [나가기]     │
└─────────────────────────────────┘
```
- Step 4(처리 중) 단계에서 이탈 시: 다이얼로그 없이 바로 이탈 허용
  - 이유: 처리는 서버에서 진행 중 — 앱을 나가도 처리는 계속됨을 명시
  - 대신: "처리가 완료되면 알림으로 알려드릴게요. 알림을 켜두시겠어요?"

#### PASS 인증 복귀 처리

```

  → PASS 앱 실행 (딥링크)
    → PASS 인증 완료
      → 진짜집 복귀 콜백 (/verify?step=2&token=xxx)
        → Step 2 자동 진입
        
[PASS 앱 미설치 시]

┌─────────────────────────────────────────┐

│  바로 알려드립니다.                     │
│                                         │
│  [알림 허용]       [그냥 나가기]       │
└─────────────────────────────────────────┘

알림 허용 후:
  → 홈으로 이동
  → 처리 완료 시 푸시 알림: "인증 완료! 등급을 확인해보세요"
  → 알림 탭 → /verify?step=5 로 복귀

재진입 경로 (/verify 탭 재방문 시):
  → "지난번에 진행하던 인증이 있습니다. 이어서 하시겠어요?"
  → [이어서 하기] → 마지막 단계로 직진
```

## 7. 접근성 체크리스트 (WCAG 2.1 AA)

### 배지 색+형태 이중코드 (색맹 대응)

| 배지 | 색 | 형태 | 텍스트 대안 |
|---|---|---|---|
| Gold | 초록 (#2D9D78) | 방패 + 체크마크 | "Gold · 실거주 인증" |
| Silver | 노랑/골드 (#D4A017) | 카메라핀 | "Silver · 현장 인증" |
| Unverified | 회색 (#9CA3AF) + 앰버 텍스트 | 점선 원 | "미인증 · 주의 필요" |

- 색 대비비: 텍스트 4.5:1 이상 (WCAG AA), 아이콘 3:1 이상 확인 필요
- 스크린리더: `aria-label="Gold 배지, 실거주 인증, 82점"` 형식

### 터치 타깃

- 모든 인터랙티브 요소 최소 44×44px (iOS HIG 기준)
- 매물 카드: 카드 전체가 탭 영역 (작은 요소 탭 오류 방지)

### 신뢰 스코어 막대 분해

- 막대 그래프에 `role="progressbar"`, `aria-valuenow`, `aria-valuemax` 부여
- 스크린리더: "세입자 사진 항목, 35점 중 35점 획득"으로 읽히도록

### 오류 메시지

- PASS 인증 실패: "본인인증에 실패했습니다. 이유: [구체적 사유]. 다시 시도하거나 고객센터에 문의하세요."
- 사진 업로드 실패: "사진 업로드에 실패했습니다. 파일 크기는 20MB 이하, JPG/PNG 형식을 사용해주세요."
- 모든 오류: 아이콘 + 텍스트 조합, 복구 방법 제시

## 8. 인터랙션 노트

### 배지 애니메이션
- 결과 화면에서 배지 등급 획득 시: 스케일 0.8 → 1.0 + 페이드인 (200ms, ease-out)
- Gold 획득 시 추가: 컨페티 파티클 효과 (강화 피드백, 1회성)

### 스텝바 트랜지션
- 스텝 완료 시: 원형 → 체크마크 채움 애니메이션 (150ms)
- 다음 스텝 활성화: 슬라이드인 (200ms, ease-in-out)

### 신뢰 스코어 막대
- 상세 화면 진입 시: 0 → 현재값으로 막대 채움 애니메이션 (600ms, ease-out)
- pending 항목: 빈 막대 + 펄스(pulse) 애니메이션으로 "대기 중" 인지

### 무한스크롤
- 커서 페이지네이션 — 다음 페이지 로딩 중: 하단 스켈레톤 카드 3개
- 모든 매물 로드 완료: "서울 관악·마포의 모든 매물을 보셨습니다." 문구

### 바텀시트 (신고)
- 진입: 아래서 위로 슬라이드 (250ms, spring 커브)
- 배경: 스크림(dimmed overlay) 50% 불투명도
- 닫기: 하단으로 스와이프 또는 배경 탭

### Before/After 프리뷰
- 수직 슬라이더 컨트롤로 두 이미지 비교 가능 (드래그)
- 기본: 좌우 50/50 분할

## 10. 제약사항 및 핸드오프 메모

### 프론트엔드팀 전달 사항
- 컴포넌트: `TrustScoreBadge`, `ListingCard`, `PhotoUploader`, `ReportSheet`, `VerificationStepper`
- 매물 상세(`/listings/[id]`)는 RSC — 초기 로딩 스켈레톤 필요
- 신뢰순 정렬은 서버 사전계산 값 사용 — 프론트 정렬 로직 구현 금지
- 낙관적 UI 금지 — 서버 확정 결과만 표시 (업로드 결과, 신고 처리 등)
- PASS 인증: 딥링크 콜백 URL 처리 필요 (`/verify?step=2&token=`)

### 백엔드팀 전달 사항
- `pending` 항목: 점수 0이 아닌 별도 `null` 또는 `"pending"` 상태값으로 내려줄 것
- 처리 완료 → 프론트 Realtime 구독 또는 폴링으로 Step 5 자동 전환
- 신고 접수 즉시: 매물 상태를 `reported`로 변경 → 검색 리스트 제외
- 결과 화면 피드백용: `score_delta`(이번 업로드로 오른 점수)와 `badge_achieved` 응답 필요

### 디자인팀 내부 다음 단계
- UI 스펙에서 Tailwind 토큰 정의 필요: Gold(#2D9D78), Silver(#D4A017), Unverified(#9CA3AF + Amber)
- 배지 아이콘 SVG: 방패+체크 / 카메라핀 / 점선원 제작
- Before/After 슬라이더 컴포넌트 디자인
