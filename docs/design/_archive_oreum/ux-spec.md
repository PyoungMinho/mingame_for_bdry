## 1. 타겟 페르소나 3종

### P1. 개발자 · 데브 (Developer Dave)

- 나이/직군: 28~40세, 스타트업 / 에이전시 백엔드·풀스택 개발자
- 방문 동기: "GPT-4o vs Claude 3.5 Sonnet — 코딩 작업에 어떤 게 빠르고 싸지?" 실용적 비교
- 검색 패턴: "claude vs gpt coding benchmark", "cheapest llm api 2026" 구글 직접 검색 → SEO 유입
- 핵심 니즈:
  - 가격/토큰 비용 한 줄로 확인 ($/1M 토큰)
  - 코딩 벤치마크 (HumanEval / SWE-Bench) 수치 직접 비교
  - API 레이트 리밋, 컨텍스트 창 크기
- 허용 인내심: 낮음. 5초 이내 숫자를 눈에 담지 못하면 뒤로가기
- UX 요구사항: 표(매트릭스) 중심 레이아웃, 숫자 강조, 필터 빠른 접근

### P2. 마케터 / 콘텐츠 기획자 (Marketing Maya)

- 나이/직군: 25~38세, 중소기업 마케팅 팀 또는 1인 크리에이터 에이전시
- 방문 동기: "카피라이팅이나 블로그 초안 쓸 때 어떤 AI 써야 해?" 위저드 유입 (SNS 공유 링크)
- 검색 패턴: "AI 카피라이팅 도구 추천", "ChatGPT 대안" — 한국어 검색 다수
- 핵심 니즈:
  - 기술 벤치마크보다 "이 작업엔 이 모델"의 추천 로직
  - 무료/유료 플랜 경계 명확화 (API 비용이 아닌 서비스 플랜 기준)
  - 공유 가능한 결과 ("내가 찾은 최적 AI" SNS 카드)
- 허용 인내심: 중간. 위저드 3단계를 기꺼이 거침
- UX 요구사항: 위저드 플로우, 쉬운 언어, 공유 CTA 강조

### P3. 일반 파워유저 (Power User Pat)

- 나이/직군: 22~45세, AI 도구 얼리어답터. 직군 불문. 레딧/X(트위터) 활동
- 방문 동기: "GPT-5 나왔다는데 Claude Opus 5랑 벤치마크 비교" — 비교 URL 직접 타이핑 또는 공유 링크
- 검색 패턴: 모델명 구체 입력, 리더보드 직접 탐색
- 핵심 니즈:
  - 전체 리더보드 한눈에 파악 (Elo 점수, 카테고리별 순위)
  - 출처/신선도 메타데이터 신뢰 확인
  - 비교 URL을 북마크/공유해서 "이게 내 레퍼런스"로 고정
- 허용 인내심: 높음. 상세 벤치마크 문서까지 읽음
- UX 요구사항: 깊은 데이터 계층, 출처 표기, permalink 공유 CTA

## 2. 핵심 사용자 플로우

### Flow A. 랜딩 → 비교 (P1·P3 메인 동선)

```
[구글 검색: "claude vs gpt"]
    |
    v
[홈 랜딩 /]
  └─ Hero 검색창에 "claude" 입력
  └─ 드롭다운: claude-3-5-sonnet / claude-opus-5 / claude-3-haiku
    |
    v
[모델 선택 1개 → 비교 추가 버튼 클릭]
    |
    v
[비교 페이지 /compare/claude-3-5-sonnet_vs_gpt-4o]
  └─ 2~4개 사이드바이사이드 매트릭스
  └─ 공유 카드 CTA ("이 비교 공유하기")
    |
    [종료 또는 모델 상세 클릭]
    v
[모델 상세 /models/claude-3-5-sonnet]
```

단계 수: 3탭 이내 비교 완료

### Flow B. 단일 모델 SEO 유입 (P1·P3 메인 동선)

```
[구글 검색: "gpt-4o specs benchmark"]
    |
    v
[모델 상세 /en/models/gpt-4o]  ← SSG 직접 착지
  └─ EntityHero: 모델명 + 출시일 + 가격 요약
  └─ 핵심 벤치마크 수치 (MMLU, HumanEval 등)
  └─ "이 모델과 비교하기" 카드 섹션
    |
    v
[비교 페이지 /compare/gpt-4o_vs_claude-3-5-sonnet]
```

단계 수: 2탭 비교 착지

### Flow C. 위저드 유입 (P2 메인 동선)

```
[SNS 공유 링크 또는 홈 "어떤 AI가 맞을까?" 버튼]
    |
    v
[위저드 1단계 /find — 직무 선택]
  └─ 개발자 / 마케터 / 디자이너 / 학생 / 기타
    |
    v
[위저드 2단계 — 작업 선택]
  └─ (직무에 따라 활성 셀 다름 → 섹션 5 매트릭스 참조)
    |
    v
[위저드 3단계 — 예산 선택]
  └─ 무료 API / $10 이하/월 / $50 이하/월 / 무제한
    |
    v
[위저드 결과 — Top 3 추천]
  └─ 추천 이유 1줄 + "상세 보기" 링크
  └─ "이 결과 공유하기" OG 카드 CTA
```

단계 수: 3탭(질문) + 결과 = 4화면, 모든 단계 뒤로가기 가능

## 3. 정보 구조 (IA)

### 사이트맵

```
/ (루트 — en 기본)
|
+-- /[locale]/                          홈 (랜딩)
|
+-- /[locale]/models/                   모델 목록 (리스트)
|   +-- /[locale]/models/[slug]         모델 상세
|
+-- /[locale]/compare/                  비교 (직접 접근 없음, 페어로만)
|   +-- /[locale]/compare/[pair]        비교 페이지
|                                        pair = slug_a_vs_slug_b (알파벳 오름차순 정렬 강제)
|
+-- /[locale]/find                      위저드 (3단계 질문 → 추천)
|
+-- /[locale]/leaderboard/              리더보드 목록 (카테고리 링크)
|   +-- /[locale]/leaderboard/[metric]  개별 리더보드
|                                        metric 예: coding / reasoning / image / price
|
+-- /[locale]/admin                     관리자 (Supabase Auth 단일, 비공개)
|
+-- /sitemap.xml                        자동 생성
+-- /robots.txt
```

### URL 규칙

| 패턴 | 예시 | 비고 |
|---|---|---|
| `/[locale]/models/[slug]` | `/en/models/gpt-4o` | slug = 케밥케이스, 공식 모델명 기준 |
| `/[locale]/compare/[pair]` | `/en/compare/claude-3-5-sonnet_vs_gpt-4o` | 두 slug 알파벳 오름차순, `_vs_` 구분자. 순서 틀리면 301 리다이렉트 |
| `/[locale]/leaderboard/[metric]` | `/en/leaderboard/coding` | metric: coding / reasoning / writing / image / price / overall |
| `/[locale]/find` | `/ko/find` | 위저드, 쿼리스트링으로 상태 유지 (`?job=developer&task=coding`) |
| `/[locale]/admin` | `/en/admin` | 크롤러 차단 (noindex, robots.txt) |

### 로케일 규칙

- 기본 로케일: `en` (path prefix 생략 가능 — `/models/gpt-4o` = `/en/models/gpt-4o`)
- 한국어: `/ko` prefix 명시 필수
- hreflang: 모든 SSG 페이지에 `<link rel="alternate" hreflang="ko">` 삽입
- 한국어 번역 범위: 홈 / 위저드 / 모델 상세(핵심 5종) / 리더보드 / 비교 → 인간 번역. 나머지 → 자동 번역 or en fallback

### 글로벌 내비게이션 구조

```
[헤더 고정]
  로고         검색창 (전역)      [리더보드]  [위저드: AI 찾기]  [언어 토글 EN|KO]  [다크모드 토글]

[푸터]
  About / Data Sources / Changelog / Admin (노출 최소화)
  © 2026 | GitHub (YAML PR 링크)
```

## 9. 접근성 체크리스트 (WCAG 2.1 AA 핵심 5개)

### A1. 색상 대비

- 일반 텍스트: 최소 **4.5:1** (배경 대비)
- 대형 텍스트 (18px 이상 / bold 14px 이상): 최소 **3:1**
- 적용 범위: 라이트·다크 모드 양쪽 검증
- 출처 신뢰도 기호(●/○/△)는 컬러만 사용 금지 → 기호 + 텍스트 병행 필수

### A2. 키보드 탐색

- 모든 인터랙티브 요소(링크, 버튼, 탭, 폼)에 Tab 포커스 도달 가능
- 포커스 링: 2px 실선, 브랜드 컬러 (기본 파란 계열, 다크모드 대비 확인)
- 위저드 카드 선택: Enter/Space 키 동작
- 비교 매트릭스: ←→ 방향키로 컬럼 탐색

### A3. 스크린리더 지원

- 모든 아이콘 버튼: `aria-label` 필수 ("비교에 추가 — GPT-4o")
- 데이터 테이블: `<table>` + `<caption>` + `scope="col"` 헤더
- 위저드 진행 상태: `role="progressbar"` + `aria-valuenow` / `aria-valuemax`
- 모달/드롭다운: `aria-expanded`, `aria-controls`, `role="dialog"` 적용

### A4. 텍스트 크기 및 반응형

- 브라우저 기본 폰트 크기 200% 확대 시 콘텐츠 겹침/잘림 없음
- 모바일(375px)에서 비교 매트릭스: 가로 스크롤 허용 + 좌측 항목 열 고정 (sticky)
- `rem` 단위 사용 (절대 `px` 사용 자제)

### A5. 오류 인식 및 복구

- 폼 에러: 붉은 테두리 + 에러 아이콘(X) + 인라인 에러 텍스트 3중 표현 (색상만 사용 금지)
- 에러 메시지: `role="alert"` 또는 `aria-live="polite"` 적용
- 관리자 폼 필수 항목: `aria-required="true"` + 시각적 asterisk(`*`) 병행
- 저장 성공/실패 토스트: 충분한 노출 시간 (최소 4초) + 수동 닫기 버튼 제공

## 10. UI 디자이너 핸드오프 노트

### 확정 사항 (UX 설계에서 결정됨)

1. **레이아웃 기준**: 데스크톱 1280px 기준, 최대 콘텐츠 폭 1200px. 모바일 375px 반응형.
2. **다크모드**: 필수. OS 시스템 설정 감지 (`prefers-color-scheme`) + 수동 토글 공존.
3. **모델 아이디덴티티**: 로고 이미지 없음. 텍스트 마크(제공사명 + 모델명 타이포그래피)만 사용. 저작권 안전 원칙.
4. **비교 매트릭스 우위 표시**: ★ 기호 + 굵은 글씨 (색상만 사용 금지 — a11y 원칙).
5. **광고 슬롯**: 6종 위치 확정 (섹션 7). 디자인 시스템에 `AdSlot` 컴포넌트로 추상화.

### UI 디자이너 결정 필요 사항

1. **브랜드 컬러 시스템**: Primary / Accent / 신뢰도 레벨(high/mid/low) 3색 정의. 라이트·다크 모드 토큰 쌍. "AI 도구" 특성상 테크-뉴트럴한 컬러 계열 권장 (딥네이비 또는 슬레이트 그레이 베이스 제안).
2. **비교 매트릭스 우위 하이라이트 방식**: ★ 기호 외에 배경색 강조 여부. 배경색 사용 시 다크모드 대비비 재검증 필요.
3. **위저드 직무 카드 아이콘**: 텍스트 이모지(MVP 임시) vs 커스텀 SVG 아이콘. 브랜드 아이덴티티 일관성 위해 커스텀 SVG 5종 권장.
4. **SourceBadge 컴포넌트 비주얼**: 신뢰도 기호(●/○/△)의 크기·컬러·타이포 디테일. 테이블 내 배치 시 가독성 확인.
5. **Changelog 섹션 비주얼**: 홈 하단 타임라인 형태 vs 단순 리스트. 시각 계층 처리 방향.
6. **OG 카드 3종 디자인**: 비교 결과 / 위저드 결과 / 모델 단일 — 각 카드의 정보 밀도와 비율 확정 후 `@vercel/og` 구현 인계 필요.
