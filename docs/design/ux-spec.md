## 1. 사용자 페르소나

### 페르소나 A — 핵심 타겟: "소확행 기록러" 지은 (27세, 직장인)
- **상황**: 매일 퇴근 후 인스타에 사진과 짧은 글을 올리지만, 시각적으로 더 특별한 무언가가 없어서 아쉽다
- **목표**: 오늘 있었던 일을 예쁘게 만화로 남기고 싶다. 내 이야기가 콘텐츠가 되길 원한다
- **불안**: "그림을 못 그리는데 만화를 만들 수 있을까?", "오래 기다려야 하면 귀찮다"
- **행동 패턴**: 모바일 100%, 짧은 집중 시간, 완성된 결과를 즉시 공유하고 싶어 함
- **결제 심리**: 월 1만원 이하 구독은 심리적 부담 낮음. "특별한 날"엔 충동 결제 가능

### 페르소나 B — 보조 타겟: "덕질 기록러" 민준 (22세, 대학생)
- **상황**: 여행·콘서트·일상을 만화로 기록하고 팬레터 대신 만화 형식으로 공유하고 싶다
- **목표**: 특정 이벤트만 만화로 만들고 싶다 (매일은 아님). 크레딧 팩 유형
- **불안**: "내 아바타가 매번 달라 보이면 일기 시리즈가 안 된다"
- **행동 패턴**: 몰아서 여러 편 생성, 저장보다 즉시 공유 지향

### 페르소나 C — 잠재 타겟: "노트 대신 만화" 혜린 (35세, 육아 중)
- **상황**: 아이 성장 일기를 예쁘게 남기고 싶지만 시간이 없다
- **목표**: 짧은 글을 넣으면 알아서 만들어 주는 서비스. 복잡하지 않아야 함
- **불안**: "너무 어려우면 못 쓴다", "개인정보(아이 일기)가 걱정된다"
- **행동 패턴**: 저녁 육아 후 10분 내 작업, 장기 아카이브 관심 높음

### 네비게이션 구조

**하단 탭바 (인증 후 고정 — 4탭)**
```
[홈]  [작성+]  [아카이브]  [마이]
```
- 작성+ 탭은 항상 중앙 강조 (FAB 스타일, 탭바 위로 돌출)
- 생성 대기 / 편집 / 공유 화면은 탭바 숨김 (풀스크린 모달)

## 3. 핵심 사용자 플로우

### 플로우 A — 신규 첫 만화 생성 (신규 유저, 무료 첫 경험)

```
목표: 처음 온 사용자가 가입부터 첫 만화 완성까지 3분 내 달성
핵심 지표: 온보딩 완료율, 첫 만화 생성 완료율
```

| 단계 | 화면 | 사용자 행동 | 감정선 | 이탈 위험 |
|------|------|------------|--------|-----------|
| 1 | 랜딩 | 예시 만화 캐러셀 스와이프, 가치 확인 | 호기심 "나도 이게 돼?" | 가입 장벽 |
| 2 | 소셜 로그인 | 구글/카카오 1탭 | 설렘 | 이메일 폼 요구 시 이탈 |
| 3 | 아바타 초기 설정 | 8종 중 1개 선택, 헤어컬러 선택 | 몰입 "내 캐릭터다" | 옵션 과부하 |
| 4 | 일기 작성 | 첫 일기 입력 (50~300자 유도) | 약간 긴장 "뭘 써야 하지?" | 빈 화면 공포 |
| 5 | 화풍 선택 | 4종 미리보기 중 선택 | 기대감 | — |
| 6 | 생성 CTA | [만화로 만들기] 탭 | 설렘 최고조 | — |
| 7 | 생성 대기 | 1컷부터 나타나는 것 구경 | 흥분 "우와!" | 60초 이탈 위험 (최대) |
| 8 | 결과 확인 | 4컷 완성 카드 전체 확인 | 감동 "이게 내 이야기야" | — |
| 9 | 공유 | 인스타 스토리/피드에 공유 | 자랑욕구 | 비율 선택 혼란 |

**이탈 위험 포인트 대응**
- 단계 2: 이메일 폼 불필요. 소셜 로그인만 제공
- 단계 3: 기본값 선택 상태로 진입. 스킵 불가 대신 "나중에 바꿀 수 있어요" 안내
- 단계 4: 플레이스홀더로 예시 일기 제공 ("오늘 퇴근길에 갑자기 비가 쏟아졌다...")
- 단계 7: SSE 1컷씩 노출 + 카운트다운 + 팁 메시지 로테이션 (상세는 §5)
- 단계 9: Web Share API 기본, 인스타 fallback 자동 처리 (상세는 §7)

### 플로우 B — 무료 한도 소진 → 유료 전환

```
목표: 한도 소진 시 구독 전환 마찰 최소화, 강압 아닌 자연스러운 업셀
핵심 지표: 한도 소진 후 전환율, 전환 시 이탈 단계
```

| 단계 | 화면 | 사용자 행동 | 감정선 | 이탈 위험 |
|------|------|------------|--------|-----------|
| 1 | 일기 작성 | 글 작성 완료 후 생성 시도 | 기대감 | — |
| 2 | 한도 소진 바텀시트 | "오늘 만화 1편을 모두 썼어요" 안내 | 아쉬움 | 강제 업셀 느낌 시 이탈 |
| 3 | 소프트 넛지 | "내일 다시 오거나, 더 많이 만들 수 있어요" + 요금제 카드 | 고민 | 가격 정보 부족 시 이탈 |
| 4 | 요금제 비교 | 베이직/프로/크레딧팩 3개 카드 | 비교 검토 | 가격 복잡성 |
| 5 | 크레딧팩 선택 | "지금 당장 1컷만" 충동형 선택 가능 | 결정 | 결제 진입 장벽 |
| 6 | 결제 완료 | 결제 후 즉시 생성 재개 | 만족 | — |
| 7 | 생성 재개 | 끊겼던 자리에서 바로 생성 시작 | 쾌감 "역시 했길 잘했다" | — |

**설계 원칙**
- 한도 소진은 **생성 시도 직전 단계**에서 차단 (글 쓰고 나서 차단 → 좌절감)
- 일기 작성 화면 상단에 잔여 횟수 표시 (오늘 1/1 사용 or "베이직: 28컷 남음")
- 바텀시트는 강제 차단이 아닌 **정보 제공형** — "내일 다시 오기", "업그레이드", 두 선택지 모두 유효
- 크레딧팩은 "지금 바로 만들기" 레이블로 충동 심리 자극
- 결제 완료 직후 → 중단했던 작성 화면으로 딥링크 복귀 (생성 즉시 재개)

## 5. 생성 대기 화면 상세

### 5.1 이탈 방지 핵심 전략

생성 소요 시간 10~60초는 모바일 사용자 이탈 임계치(8초)를 크게 초과한다.
이탈 방지는 세 가지 레이어로 설계한다.

**레이어 1: 즉각적 피드백 (SSE 스트리밍)**
- 생성 요청 직후 0~3초 내 첫 컷 노출 시작
- 4컷 그리드 = 완성 컷부터 순차 채워짐 (1컷 → 2컷 → 3컷 → 4컷)
- 각 컷 완성 시: 스켈레톤 → 이미지 fade-in (400ms)
- 컷 생성 중 스켈레톤: 가로선 shimmer 애니메이션

**레이어 2: 진행 인식 (카운터 + 팁)**

```
상단 진행률 바:
- 0%: 생성 요청 즉시
- 25%: 컷 1 완성
- 50%: 컷 2 완성
- 75%: 컷 3 완성
- 100%: 컷 4 완성 → 결과 화면 자동 전환

팁 메시지 로테이션 (8초마다 fade 교체):
T0: "내 일기가 4컷으로 재탄생하고 있어요 ✍"
T8: "화풍을 캘리브레이션 중이에요. 조금만 기다려요"
T16: "아바타 캐릭터 일관성 확인 중... 4컷 모두 같은 내 얼굴로!"
T24: "완성된 만화는 아카이브에 자동 저장돼요"
T32: "1년이 지나면 365편이 쌓여요. 오늘이 그 시작!"
T40: "클로즈드 베타 참여 감사해요. 빠른 생성 중..."
```

**레이어 3: 백그라운드 허용 + 알림**

```
백그라운드 전환 배너 (생성 시작 5초 후 표시):
┌─────────────────────────────────────┐
│ 다른 앱을 써도 돼요                   │
│ 완성되면 알림을 보내드릴게요   [허용]  │
└─────────────────────────────────────┘

- [허용] 탭: 브라우저 알림 권한 요청 (Web Push API)
- 권한 허용 시: 생성 완료 후 Push Notification 발송
- 권한 거부 시: 배너 숨김 (강제 없음)
- 알림 탭 → 앱 포그라운드 복귀 → 결과 화면 딥링크
```

## 7. 공유 플로우

### 7.1 마찰 제로 설계 원칙

공유 화면에서 SNS 앱까지 2탭 이내 완료 목표.
각 플랫폼별 최적화된 fallback 동선 설계.

### 7.2 공유 타깃별 동선

**인스타그램**

```
[인스타] 탭
  → Web Share API 지원 브라우저 (iOS Safari 15+, Android Chrome 86+)
     └── navigator.share({ files: [imageFile] })
         └── 네이티브 시트 → 인스타 선택 → 스토리/피드 선택
  → Web Share API 미지원 or 인스타 미설치
     └── 이미지 자동 클립보드 복사
         └── 안내 바텀시트: "이미지가 복사됐어요! 인스타에서 붙여넣기 하세요"
         └── [인스타 앱 열기] 딥링크 버튼 (instagram://camera)
```

**X (트위터)**

```
[X] 탭
  → Web Share API로 이미지 + 텍스트 공유
  → Fallback: twitter.com/intent/tweet?text=...&url=... (이미지 미첨부)
     └── 안내: "X 앱에서 이미지를 추가해 주세요"
```

**카카오톡**

```
[카카오] 탭
  → 카카오 JS SDK (KakaoTalk.sendDefault)
     └── 4컷 이미지 + 공유 링크 카드 형식
  → SDK 미로드 시: 카카오톡 앱 딥링크 + 이미지 복사 fallback
```

**링크 복사**

```
[링크복사] 탭
  → toonlog.kr/share/:id (OG 메타 포함 공유 페이지)
  → Clipboard API 복사 완료 토스트: "링크 복사됐어요!"
  → 3초 후 토스트 사라짐
```

### 7.3 비율별 Satori 렌더링

| 비율 | 해상도 | 주 용도 | 로딩 시간 |
|------|--------|---------|-----------|
| 1:1 | 1080×1080 | 인스타 피드 | 기본 (미리 생성) |
| 16:9 | 1920×1080 | X/트위터 | 탭 시 On-demand |
| 9:16 | 1080×1920 | 릴스/스토리 | 탭 시 On-demand |

- 결과 화면 진입 시 1:1 미리 생성 시작
- 16:9, 9:16는 탭 시 On-demand (스켈레톤 → 렌더 완료까지 최대 3초)
- 로딩 중 공유 버튼 비활성화 + 스피너

```
┌─────────────────────────────────┐
│  이런, 생성에 실패했어요         │  ← 제목
│  일시적인 문제일 수 있어요.       │
│  작성한 일기는 저장됐어요.        │  ← 안심 메시지 (데이터 손실 없음 강조)
├─────────────────────────────────┤
│  [다시 시도하기]                 │  ← Primary CTA (한도 차감 없음 명시)
│  한도는 차감되지 않아요          │  ← 중요 안심 문구
├─────────────────────────────────┤
│  [나중에 할게요]                  │  ← 초안으로 저장, 홈 복귀
└─────────────────────────────────┘
```

### 8.4 모더레이션 차단 (부적절 콘텐츠)

```
┌─────────────────────────────────┐
│  이 일기는 만화로 만들 수 없어요 │
├─────────────────────────────────┤
│  일기 내용이 서비스 이용 정책에  │
│  맞지 않아 생성이 제한됐어요.   │
│  내용을 수정하고 다시 시도해 보세요│
├─────────────────────────────────┤
│  [일기 수정하기]                 │  ← 작성 화면으로 복귀
│  [이용정책 보기]                 │  ← 텍스트 링크
└─────────────────────────────────┘

주의: 차단 이유 구체적 명시 금지 (우회 방지)
한도는 차감되지 않음 (명시)
```

### 8.5 네트워크 끊김

```
생성 중 오프라인 감지:
┌─────────────────────────────────┐
│  인터넷 연결이 끊어졌어요        │
│  연결이 복구되면 자동으로 재시도해요│
│  ████████░░░░░░░░  대기 중...   │  ← 대기 상태 표시
└─────────────────────────────────┘

공유 중 오프라인 감지:
- 공유 버튼 비활성화
- 토스트: "인터넷 연결을 확인해 주세요"
- 이미지 저장은 오프라인에서도 가능 (로컬 처리)
```

### 8.6 미성년자 / 보호자 동의 플로우

```
회원가입 시 생년월일 입력:
- 만 14세 미만 → 보호자 동의 화면 (이메일 인증)
- 만 14~18세 → 청소년 이용 가능 확인 체크박스
- 만 19세 이상 → 일반 가입

보호자 동의 화면:
┌─────────────────────────────────┐
│  보호자 동의가 필요해요          │
│  서비스를 이용하려면             │
│  보호자의 동의가 필요해요.        │
├─────────────────────────────────┤
│  보호자 이메일   [___________]  │
│  [동의 이메일 발송]              │
└─────────────────────────────────┘
```

### 8.7 아바타 캐릭터 일관성 붕괴 (얼굴 임베딩 실패)

```
사용자에게는 노출하지 않음.
백엔드 자동 재생성(최대 2회) 로직이 처리.
2회 후에도 실패 시 → §8.3 생성 실패 플로우.
생성 대기 화면에서는 "캐릭터 다듬는 중이에요" 팁 메시지로 대체.
```

## 9. 마이크로카피 가이드

### 9.1 톤 원칙

- **친근하고 따뜻하게**: "오류가 발생했습니다" → "이런, 뭔가 잘못됐어요"
- **사용자 편 들기**: 문제가 생겼을 때 사용자 탓 없이 해결책 제시
- **기대감 증폭**: 생성 중 / 완성 직전에 설렘 유발
- **부담 없이**: 한도, 결제, 정책 안내는 강압이 아닌 정보 제공
- **짧고 명확하게**: 1문장 이내, 2문장 초과하지 않기

### 9.2 화면별 카피 예시

**랜딩**
```
H1: "누구나 만화 작가가 될 수 있다"
H2: "오늘 일기를 4컷 만화로"
CTA: "무료로 시작하기"
부제: "가입 후 바로 첫 만화를 만들 수 있어요"
```

**일기 작성**
```
레이블: "오늘은 어떤 일이 있었나요?"
플레이스홀더 (월요일): "월요일이라 조금 힘들었나요? 그래도 오늘 하루 어땠나요?"
플레이스홀더 (금요일): "드디어 금요일! 오늘 하루를 만화로 남겨봐요"
글자수 부족: "조금 더 적어주면 더 풍부한 만화가 돼요 (최소 50자)"
글자수 초과: "너무 길어요. 핵심만 남겨볼까요?"
CTA: "만화로 만들기 →"
```

**생성 대기**
```
헤더: "만화 만드는 중..."
서브: "내 이야기가 4컷이 되고 있어요"
팁1: "완성되면 알림으로 알려드려요. 다른 앱 써도 돼요!"
팁2: "4컷 만화는 아카이브에 자동으로 저장돼요"
팁3: "1년이 쌓이면 나만의 만화 앨범이 완성돼요"
```

**생성 완료**
```
완성 애니메이션: "완성됐어요! ✨"
서브: "내 이야기가 만화가 됐어요"
편집 버튼: "말풍선 고치기"
공유 버튼: "공유하기"
```

**한도 소진**
```
제목: "오늘 만화 1편을 다 썼어요"
서브: "내일 또 만들 수 있어요. 아니면 더 많이 만들 수 있는 방법도 있어요"
업셀 CTA: "더 만들기"
거절: "내일 다시 올게요"
```

**에러/실패**
```
생성 실패: "이런, 생성에 실패했어요. 작성한 일기는 저장됐으니 걱정 마세요"
재시도: "다시 시도하기 (한도 차감 없어요)"
모더레이션: "이 일기는 만화로 만들 수 없어요. 내용을 조금 바꿔볼까요?"
네트워크: "인터넷 연결을 확인해 주세요"
```

**공유**
```
링크 복사 완료: "링크 복사됐어요!"
인스타 fallback: "이미지가 복사됐어요! 인스타에서 붙여넣기 하세요"
AI 고지: "이 만화는 AI가 생성했습니다"
```

**아카이브**
```
스트릭 배너 (7일): "7일 연속 기록 중! 이 페이스면 100일도 금방이에요"
스트릭 배너 (30일): "30일 연속! 벌써 한 달치 만화가 쌓였어요"
빈 상태: "아직 만화가 없어요. 오늘 첫 만화를 만들어볼까요?"
```

## 10. 접근성 체크리스트

### 10.1 색상 대비 (WCAG 2.1 AA 기준)

| 항목 | 기준 | 확인 방법 |
|------|------|----------|
| 일반 텍스트 (16px+) | 대비율 4.5:1 이상 | UI팀이 hex 값으로 검증 |
| 대형 텍스트 (24px+, bold 18px+) | 대비율 3:1 이상 | |
| UI 컴포넌트 경계선 | 대비율 3:1 이상 | 버튼 테두리, 입력 필드 |
| 다크모드 별도 검증 | 동일 기준 적용 | 다크 배경 기준 재검증 |
| 워터마크 텍스트 | 이미지 위 가독성 | 반투명 배경 필수 |

### 10.2 폰트 크기

| 요소 | 최소 크기 | 권장 |
|------|-----------|------|
| 본문 텍스트 | 16px | 16px |
| 보조 텍스트 (Caption) | 12px | 14px |
| 레이블 | 14px | 16px |
| 버튼 텍스트 | 16px | 16px |
| 말풍선 텍스트 | 14px | 16px |

> 시스템 폰트 크기 설정 따름: `font-size: 1rem` 기준, rem 단위 사용

### 10.3 터치 타깃

| 요소 | 최소 크기 | 비고 |
|------|-----------|------|
| 버튼 (Primary/Secondary) | 44×44px | 권장 56px |
| 아이콘 버튼 | 44×44px | 투명 패딩으로 확장 |
| 아바타 프리셋 그리드 | 64×64px | 충분 |
| 캘린더 날짜 셀 | 44×44px | 최소 준수 |
| 말풍선 핸들 | 44×44px | 투명 오버레이 |
| 꼬리 방향 버튼 | 44×44px | 8개 모두 |
| 헤더 뒤로가기 | 44×44px | |

### 10.4 스크린리더 (ARIA)

| 화면 | 필수 ARIA 처리 |
|------|--------------|
| 일기 작성 | `<textarea aria-label="일기 내용" aria-describedby="char-count">` |
| 글자수 | `<span id="char-count" aria-live="polite">75 / 300</span>` |
| 생성 대기 | `<div role="progressbar" aria-valuenow="50" aria-valuemin="0" aria-valuemax="100">` |
| 이미지 컷 | `<img alt="4컷 만화 2번째 컷: 주인공이 편의점 앞에 서 있는 장면">` (LLM 생성 alt) |
| 말풍선 편집 캔버스 | `<canvas aria-label="말풍선 편집 캔버스">` + 보조 텍스트 편집 UI |
| 탭바 | `<nav aria-label="하단 메인 내비게이션">` |
| 선택된 탭 | `aria-selected="true"` |
| 비활성 버튼 | `aria-disabled="true"` (disabled 속성과 함께) |

### 10.5 키보드 / 포커스

- 포커스 표시선: 2px solid, 대비율 3:1 이상
- 모달 / 바텀시트: 포커스 트랩 적용 (tab cycling 내부 한정)
- ESC: 모달/바텀시트 닫기
- Enter: 선택된 CTA 실행

### 10.6 모션 / 애니메이션

```css
@media (prefers-reduced-motion: reduce) {
  /* shimmer, fade, scale 애니메이션 비활성화 */
  * { animation: none !important; transition: none !important; }
  /* SSE 컷 노출은 즉시 표시 (fade 없이) */
}
```

### 10.7 기타

- 음성 입력: 화면 밖에서도 마이크 활성 표시 (상태 표시등 제공)
- 색 외 정보 전달: 글자수 초과 = 빨간색 + 경고 아이콘 + 텍스트 (색만 의존 금지)
- 한도 배지: 수치 텍스트 + 색상 병행 (색각 이상 고려)
- 스트릭 배지: 아이콘 + 숫자 + 텍스트 레이블 병행
